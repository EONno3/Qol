# 용병 이력서 제작 규격

이 문서는 실제 용병 이력서를 만들기 위한 1차 제작 규격이다.

최종 문체 확정 문서가 아니다. 목적은 현재까지 정리된 태그, 스탯, 장비, 정보 해금 규칙을 바탕으로 **플레이 테스트 가능한 용병 이력서**를 만들 수 있게 하는 것이다.

## 1. 문서 목적

용병 이력서는 플레이어에게 다음 질문을 던져야 한다.

```text
이 용병은 무엇을 잘할 것 같은가?
어떤 상황에서 위험해 보이는가?
장비/임플란트 세팅으로 보완할 수 있는가?
숨겨진 문제나 이야기 단서가 있어 보이는가?
고용할 가치가 있는가?
```

용병 이력서는 모든 정보를 공개하는 데이터 시트가 아니다.

Phase가 낮을수록 플레이어는 이력과 습관, 외형, 장비 단서를 읽고 추론해야 한다.

핵심은 플레이어가 이력서의 줄글을 읽고 “이 사람은 어떤 성격인지”, “무엇을 잘할 것 같은지”, “어떤 상황에서 위험할지”를 스스로 판단하게 만드는 것이다.

## 2. 기본 원칙

- 용병의 기본 성능은 5대 스탯이 담당한다.
- 태그는 스탯을 대체하지 않고, 스탯이 어떤 상황에서 의미 있는지 바꾼다.
- 시스템 태그와 이야기 태그를 구분한다.
- 현재 상태와 신규 부상은 어떤 Phase에서도 숨기지 않는다.
- 만성/내재 문제는 Phase에 따라 숨기거나 암시할 수 있다.
- 이야기 핵심 비밀은 Phase 2에서도 전부 공개하지 않을 수 있다.
- 이력서는 특정 미션을 위해 작성된 문서처럼 보이면 안 된다.
- 미션 적합성 해석은 이력서가 아니라 미션 슬롯 매칭 화면에서만 제공한다.
- 용병 이력서의 Phase는 **용병 분석 기관 레벨**만 따른다. 미션 분석 기관 레벨이 높아도 용병 이력서 자체의 정보가 대신 열리지는 않는다.
- 미션 슬롯 매칭의 표시 깊이는 **용병 분석 기관 레벨과 미션 분석 기관 레벨 중 낮은 쪽**을 따른다.
- 이력 텍스트는 태그 목록의 포장지가 아니라 플레이어 판단의 핵심 재료다.
- Phase 2에서도 이력서가 정답표나 미션 공략표로 바뀌면 안 된다.
- 분석 정보는 이력 텍스트를 읽고 세운 가설을 보조하거나 검증하는 역할이어야 한다.

## 2.1 이력서-미션 분리 원칙

용병 이력서는 플레이어가 미션 목록을 보기 전에도 읽는 상시 고용 정보다.

따라서 이력서는 특정 미션의 요구 조건을 미리 풀어주는 문서가 아니다.

작성 규칙:

- 이력서는 출신, 경력, 성향, 장비, 현재 상태, 과거 사건을 다룬다.
- 이력서는 "이 미션에 맞다/안 맞다"를 직접 말하지 않는다.
- 이력서 문장은 특정 미션의 목표, 장소, 위험, 장치, 적, 보상과 직결되면 안 된다.
- 특정 미션 요구와 용병 정보를 연결하는 해석은 미션 슬롯 매칭 화면에만 둔다.
- 매칭 화면은 용병 분석 기관과 미션 분석 기관이 모두 해당 단계 이상을 확보했을 때만 그 단계의 해석을 표시한다.
- Phase 2에서도 이력서는 용병의 상세 정보만 공개한다. 미션 정답표가 되면 안 된다.
- 이력서 문장은 플레이어가 성격, 장점, 위험을 읽어낼 수 있는 행동 기록과 습관 중심으로 작성한다.
- `강함`, `약함`, `적합`, `위험` 같은 판정어만 나열하지 않고, 그렇게 판단할 수 있는 근거가 되는 사건을 쓴다.

금지 예:

```text
경고가 진짜인지 아닌지 가르는 일에는 약하다.
실버램프 같은 임대 창고 보안망을 다뤄본 적이 있다.
밸브와 압력계 앞에서는 다른 용병보다 오래 버틴다.
이 용병은 이 미션의 핵심 요구를 해결할 수 있다.
```

허용 예:

```text
민간 경비 계약에서 대응 수위 문제로 재계약이 끊긴 기록이 있다.
출입 관리 하청 사무소에서 오래 일했고, 낡은 인증 기록을 정리한 경력이 있다.
정비 숙소 출신으로, 오래된 설비 소리와 압력 변화를 잘 구분한다는 평가가 있다.
명령 대기보다 현장 판단을 앞세우는 편이라는 보고가 반복된다.
```

판단 기준:

- 이력서만 따로 떼어 읽어도 특정 미션용 캐릭터처럼 보이지 않아야 한다.
- 미션 카드 없이도 자연스러운 용병 소개여야 한다.
- 미션 카드와 합쳤을 때만 적합/부적합 해석이 생겨야 한다.

- 이 용병의 위험:

- 보완 가능한 준비:

### 장비/임플란트
- 기본 무기:

## 5. 플레이어 노출 구조

플레이어에게 보이는 이력서는 Phase에 따라 달라진다.

### Phase 0 이력서

목적:

- 플레이어가 텍스트를 읽고 용병의 강점과 위험을 추론한다.
- 직접 태그명, 숨겨진 만성 상태, 정확한 시스템 효과를 숨긴다.

보여줄 것:

- 이름.
- 별칭.
- 출신 한 줄.
- 짧은 이력 텍스트 4~6문장.
- 현재 상태.
- 외형/장비의 눈에 띄는 단서.

숨길 것:

- 정확한 태그명.
- 숨겨진 만성 상태.
- 내재 징크스 이름.
- 이야기 비밀.
- 정확한 시스템 효과.
- 전체 장비/임플란트 상세.

작성 형식:

```text
## [Phase 0] 용병 이력서

- 이름:
- 별칭:
- 출신:
- 현재 상태:

### 이력
4~6문장.

### 눈에 띄는 사항
- 외형 또는 장비 단서:
- 습관 또는 말투 단서:
```

Phase 0 현재 상태 규칙:

- 미션 후 생긴 신규 부상, 피로, 파손, 노출은 직접 표시한다.
- 고용 전부터 있던 만성/내재 문제는 텍스트로만 암시한다.

### Phase 1 이력서

목적:

- 용병 분석 기관이 스탯과 대표 신호를 보여준다.
- 정답을 전부 공개하지 않고, 위험 방향을 알려준다.

보여줄 것:

- Phase 0 정보.
- 5대 스탯 요약.
- 대표 분석 3~5개.
- 미확인 위험 0~1개.
- 장비/임플란트 요약.

숨길 것:

- 정확한 숨김 태그명.
- 이야기 태그의 정체.
- 만성/내재 문제의 정확한 이름.
- 전체 태그 목록.

작성 형식:

```text
## [Phase 1] 용병 이력서

- 이름:
- 별칭:
- 출신:
- 현재 상태:

### 스탯 요약
- 프레임:
- 쿨:
- 와이어:
- 사이퍼:
- 펄스:

### 대표 분석
- 강점:
- 강점:
- 위험:
- 조건부:
- 미확인:

### 장비/임플란트 요약
- 무기:
- 임플란트:
- 가시성:

### 이력
Phase 0 이력 유지.
```

대표 분석 수 제한:

- 전체 최대 5개.
- 강점 2~3개.
- 위험 1~2개.
- 미확인 0~1개.

### Phase 2 이력서

목적:

- 시스템 판단에 필요한 정보를 명확히 공개한다.
- 후반 플레이어가 미션 매칭과 장비 세팅을 최적화할 수 있게 한다.

보여줄 것:

- 시스템 태그 상세.
- 만성 상태.
- 내재 징크스.
- 장비/임플란트 상세 요약.
- 공개 가능한 이야기 단서.

숨길 수 있는 것:

- 반전성 과거.
- 장기 분기 핵심.
- 특정 NPC와의 비밀 관계.
- 아직 발생하지 않은 추적/복수 이벤트.

작성 형식:

```text
## [Phase 2] 용병 이력서

- 이름:
- 별칭:
- 출신:
- 현재 상태:
- 계약 조건:

### 스탯
- 프레임:
- 쿨:
- 와이어:
- 사이퍼:
- 펄스:

### 보유 태그 상세 내역
- 출신/경험:
- 장비/임플란트:
- 성향/상태:
- 태생/만성 부정:
- 장비 속성:

주의:

- `보유 태그 상세 내역`에는 스탯 수치나 스탯 한계를 다시 쓰지 않는다.
- 스탯 수치는 바로 위 `스탯` 섹션에서만 보여준다.
- 스탯이 낮아서 생기는 불리함은 미션 슬롯 매칭 화면에서만 해석한다.
- 금지: `스탯 한계: Frame 36`, `전술 한계: Cypher 58`.

### 장비/임플란트
- 외장형 무기:
- 내장형 무기:
- 주요 임플란트:
- 파손/보험 리스크:

### 이력 및 세부 사항
Phase 0 이력을 확장하되, 특정 미션 해석은 쓰지 않는다.
```

## 6. 스탯 작성 기준

스탯은 용병의 기본 성능이다.

- 프레임: 육체, 하드웨어 수용성, 전투 생존, 중화기.
- 쿨: 침착함, 정신 마모 저항, 협상, 사이버싸이코 억제.
- 와이어: 네트워크, 해킹, 기계 조작, 지원.
- 사이퍼: 은닉, 흔적 제거, 현장 분석, 잠입.
- 펄스: 직관, 돌발 감지, 비정형 위기 대응.

작성 규칙:

- 스탯만으로 용병이 완성되면 안 된다.
- 태그만으로 스탯을 대체해도 안 된다.
- 낮은 스탯에도 조커 카드가 될 여지를 줄 수 있다.
- 높은 스탯에도 망설일 위험을 붙일 수 있다.
- 스탯값 자체를 태그로 만들지 않는다.
- `#낮은_Frame`, `#높은_Cypher`, `#Wire_경계값` 같은 태그는 금지한다.
- 낮은 스탯은 `Frame: 36`, `Frame (하)`, `Frame 60 미만`처럼 수치/등급/판정으로 표시한다.
- 태그는 스탯값으로 환원되지 않는 출신, 성향, 장비, 상태, 경험, 비밀에만 쓴다.

예:

```text
좋음:
Frame: 36
Cypher: 58
Frame 60 이상 권장 미충족

나쁨:
#낮은_Frame
#Cypher_경계값
#높은_Wire
```

## 7. 장비/임플란트 작성 기준

용병의 장비는 미션 판단에 직접 연결되어야 한다.

필수로 볼 것:

- 외장형 무기 여부.
- 내장형 무기 여부.
- 가시성.
- 주요 임플란트 파츠.
- 파손 리스크.
- 보험 필요성.
- 장비와 스탯 요구치.

작성 규칙:

- 장비 세부 스펙을 Phase 0에 모두 쓰지 않는다.
- 외형 단서로 가시성을 암시할 수 있다.
- 내장형 무기는 조커 카드 후보가 될 수 있다.
- 고성능 장비는 유지비, 가시성, 파손 리스크와 같이 봐야 한다.

## 8. 현재 상태 작성 기준

현재 상태는 플레이어의 즉각 판단에 중요하다.

반드시 즉시 공개:

- 신규 부상.
- 신규 피로/스트레스.
- 장비 파손.
- 임플란트 손상.
- 새로 생긴 수배/노출 상태.

숨길 수 있는 것:

- 고용 전부터 있던 만성 문제.
- 내재 징크스.
- 과거 사건의 원인.
- 이야기 비밀.

작성 규칙:

- 플레이어가 직접 만든 결과는 숨기지 않는다.
- 숨겨진 만성 문제는 Phase 0에서 이력과 습관으로 암시한다.
- Phase 1에서는 미확인 위험으로 표시할 수 있다.
- Phase 2에서는 시스템적으로 필요한 상태를 공개한다.

## 9. 시스템 태그와 이야기 단서 분리

용병 이력서에는 시스템 판단과 이야기 단서가 함께 들어간다.

하지만 둘을 한 태그에 섞지 않는다.

시스템 정보:

- 미션 적성.
- 출격 조건.
- 장비 운용.
- 상태/후유증.
- 비용/계약.
- 가시성.

이야기 정보:

- 과거.
- 비밀.
- 원한.
- 특정 NPC와의 관계.
- 특정 팩션과의 숨은 연결.
- 배신/복수/구원 이벤트.

분리 예:

```text
시스템 정보:
호흡계가 특정 환경 미션에서 위험하다.

이야기 정보:
그 문제가 특정 기업의 실험 때문일 수 있다.
```

## 10. 이력 텍스트 작성 기준

이력 텍스트는 분위기와 판단 단서를 동시에 제공해야 한다.

좋은 이력 텍스트는 다음을 한다.

- 출신과 세계관을 보여준다.
- 강점을 직접 설명하지 않고 행동으로 암시한다.
- 위험을 단정하지 않고 습관이나 기록으로 암시한다.
- 장비/임플란트의 외형적 단서를 준다.
- 이야기 단서를 남긴다.

나쁜 방향:

```text
이 용병은 Frame이 높고 호흡계 마모 태그가 있으며 협소 공간 공포가 있다.
잠입 미션에는 부적합하고 전투 미션에는 적합하다.
```

좋은 방향:

```text
하층 설비 구역에서 오래 굴렀고, 무거운 부품을 혼자 옮겼다는 기록이 많다.
말수가 적지만, 밀폐된 작업 구역 배정을 피한 흔적이 몇 차례 남아 있다.
왼팔의 구형 크롬 의수는 힘은 좋지만 검문대에서 시선을 끌기 쉽다.
```

## 11. 이력서 금지 표현

이력서에는 아래 표현을 쓰지 않는다.

- 특정 미션의 장치, 장소, 목표를 직접 언급하는 문장.
- "이 미션에 유리/불리"처럼 매칭 결과를 말하는 문장.
- "경고 판별", "밸브 개폐", "오인식별"처럼 현재 작성 중인 미션 요구를 그대로 반영한 문장.
- "전투보다 확인 절차가 강하다"처럼 미션 해결 방식을 평가하는 문장.
- Phase 2 이력 문단에서 용병 분석 기관이 현재 미션 요구에 맞춰 해석하는 문장.

대신 아래처럼 쓴다.

- 과거 계약 기록.
- 반복된 행동 습관.
- 장비 외형과 운용 방식.
- 과거 사고, 재계약 실패, 평판 기록.
- 용병이 스스로 말하는 태도.

## 12. 대표 분석 작성 기준

Phase 1의 대표 분석은 정답이 아니라 신호다.

작성 규칙:

- 최대 5개.
- 강점과 위험을 섞는다.
- 정확한 숨김 태그명을 공개하지 않는다.
- 미확인 위험은 0~1개만 둔다.
- 모든 태그를 펼치지 않는다.

권장 구조:

```text
대표 분석:
- 강점:
- 강점:
- 위험:
- 조건부:
- 미확인:
```

## 13. 미션 슬롯 매칭 작성 기준

용병을 미션에 대입했을 때는 미션 슬롯 매칭 화면에서 3줄 요약을 기본으로 한다.

이 내용은 용병 이력서에 넣지 않는다.

```text
장점:
단점:
판단:
```

작성 규칙:

- 장점은 스탯/장비/태그 중 가장 큰 이유 1~2개만 쓴다.
- 단점은 현재 미션에서 실제로 위험한 것만 쓴다.
- 판단은 "가능/위험/비추천" 같은 결론을 주되, 이유를 짧게 남긴다.
- Phase 0에서는 이 요약을 표시하지 않거나 매우 거칠게 둔다.
- Phase 1에서는 신호 중심.
- Phase 2에서는 충돌/시너지 이유를 공개한다.

## 14. 용병 유형별 작성 초점

### 전투형

우선 보여줄 긴장:

- 화력은 충분한가.
- 생존력이 있는가.
- 장비 파손과 보험 리스크가 큰가.
- 고가시성 때문에 잠입/기업 미션에 불리한가.

### 잠입형

우선 보여줄 긴장:

- 흔적을 지울 수 있는가.
- 가시성이 낮은가.
- 들켰을 때 플랜 B가 있는가.
- 순수 전투력이 부족한가.

### 지원형

우선 보여줄 긴장:

- 기술 작업을 끝낼 수 있는가.
- 소요 시간을 줄일 수 있는가.
- 실패 시 위약금이나 신뢰 하락이 큰가.
- 환경 대응 능력이 있는가.

### 기업 대응형

우선 보여줄 긴장:

- 보안 검문을 통과할 수 있는가.
- 쿨과 사이퍼가 받쳐주는가.
- 팩션 평판이 위험한가.
- 실패 시 후폭풍이 큰가.

### 비밀 대응형

우선 보여줄 긴장:

- 펄스와 쿨이 높은가.
- 미확인 위험에 버틸 수 있는가.
- 이야기 태그가 특수 반응을 만들 가능성이 있는가.
- 플레이어가 완전한 정보 없이 보낼 만한가.

## 15. 결과 보고서 연결

용병 이력서는 결과 보고서와 이어져야 한다.

결과 보고서에서 드러날 수 있는 것:

- 숨겨진 만성 문제가 처음으로 의심되는 순간.
- 조커 카드가 발동한 순간.
- 장비/임플란트가 파손된 이유.
- 특정 팩션이나 NPC가 용병에게 반응한 이유.
- 용병의 충성/불만/배신/성장 단서.

신규 상태는 결과 보고서 이후 이력서 현재 상태에 즉시 반영한다.

## 16. 작성 완료 체크리스트

용병 이력서 하나를 만든 뒤 아래를 확인한다.

- Phase 0 이력 텍스트가 정답지처럼 보이지 않는가?
- 이력서가 특정 미션에 맞춰 작성된 것처럼 보이지 않는가?
- 미션 적합/부적합 해석이 이력서가 아니라 미션 슬롯 매칭에만 있는가?
- 현재 상태와 숨겨진 만성 문제가 구분되는가?
- 스탯과 태그가 서로 역할을 나눠 갖는가?
- 시스템 태그와 이야기 단서가 섞이지 않는가?
- 대표 분석이 5개 이하인가?
- 이야기 단서가 너무 많이 노출되지 않는가?
- 장비/임플란트가 미션 판단과 연결되는가?
- 고용할 이유와 망설일 이유가 둘 다 있는가?
- 결과 보고서에서 이 용병의 특성이 설명될 수 있는가?
- 테스트 미션 슬롯에 넣었을 때 장점/단점/판단 3줄이 따로 나오는가?

## 17. 현재 단계에서 하지 않는 것

- 실제 용병 대량 작성.
- 실제 태그 대량 작성.
- 최종 문체 확정.
- 수치 밸런스 확정.
- 모든 이야기 비밀 확정.

지금은 테스트 가능한 1차 용병 이력서를 만들기 위한 규격을 잡는 단계다.
