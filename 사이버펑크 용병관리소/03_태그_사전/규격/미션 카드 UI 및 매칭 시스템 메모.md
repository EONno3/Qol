# 미션 카드 UI 레이아웃 및 동적 매칭 시스템 메모

이 문서는 `미션 카드 제작 규격`에서 분리된 **UI 레이아웃 시각화 제안** 및 용병 이력서가 슬롯에 대입되었을 때 연산되는 **동적 매칭 시스템 설계 메모**이다.

---

## 1. UI/UX 레이아웃 제안 (초보자 진입장벽 완화)

미션 카드의 정보 과밀을 해소하고 초보 플레이어가 직관적으로 위험을 판단하게 돕는 시각적 가이드라인이다.

### 1.1 핵심 속성 대표 뱃지화 (상단 고정 패널)
- 미션 카드 상단에 텍스트 대신 직관적인 **아이콘 뱃지**를 배치하여 미션의 요구 본질을 한눈에 식별하게 한다.
- **예시 뱃지 구성**:
  - `[미션 분석 뱃지]`: 현재 미션 카드/요약서에 해금된 Phase 0 / 1 / 2 단계 표시. 미션 분석 기관 레벨 또는 미션 분석 슬롯 연산 상태를 자물쇠 및 주파수 동기화 게이지로 연출.
  - `[용병 분석 뱃지]`: 현재 슬롯에 올린 용병 이력서의 Phase 0 / 1 / 2 판독 단계 표시. 용병 분석 기관 레벨 또는 용병 분석 슬롯 연산 상태를 별도 아이콘으로 연출.
  - `[매칭 해석 뱃지]`: `min(용병 분석 레벨, 미션 분석 레벨)` 기준으로 실제 표시 가능한 매칭 단계 표시.
  - `[위험 주 스탯 뱃지]`: `[FRAME]`, `[COOL]`, `[WIRE]`, `[CYPHER]`, `[PULSE]` 중 활성화된 스탯만 아이콘으로 강조.
  - `[필수 기어 주의보]`: 특정 환경 보정 장비가 필요할 시 `[필수: 호흡 필터]` 또는 `[필수: 고고도 기압 보정]` 아이콘 활성화.

### 1.2 브리핑 단서 하이라이트 (Phase 0)
- 초보 플레이어가 Phase 0 브리핑 줄글에서 힌트를 추론할 수 있도록, 기획자가 지정한 핵심 키워드에 **시각적 강조(Underline, 폰트 색상 변경 등)**를 적용한다.
- **예시**:
  - "가스가 세어 나오는 녹슨 배수 파이프가 널려 있으니, **[필터 규격]** 단단히 맞춰 가라고."
  - "피복이 벗겨진 초고압 와이어가 무방비로 널려 있으니, **[절연 보호 장비]**나 **[송전선 접지 기술]**이 없는 머저리를 보냈다간..."

### 1.3 난이도 대비 예상 손실 지표 노출
- 난이도(별 개수) 외에 용병의 건강 및 재정에 미칠 리스크 게이지를 시각화하여 숙련도를 높인다.
- **예시**:
  - **예상 누적 피로도**: `[ 안전 / 주의 / 위험 ]`
  - **예상 의체 마모 수준**: `[ 경미 / 보통 / 치명 ]`

## 2. 용병 슬롯 대입 시 동적 매칭 시스템 연산 메모

미션 카드 자체는 정적이며, 용병 이력서를 매칭 슬롯에 "대입하는 순간" 실시간으로 아래 항목들이 연산되어 매칭 UI에 출력된다.

### 2.0 매칭 해석 레벨 산정
매칭 UI는 용병 분석 기관과 미션 분석 기관의 레벨을 동시에 참조한다.

```text
매칭 해석 레벨 = min(용병 분석 레벨, 미션 분석 레벨)
```

- **매칭 Lv.0**: 둘 중 하나라도 Lv.0이면 매칭 신호를 표시하지 않는다. 이 경우 용병 이력서나 미션 카드 자체의 독립 정보만 볼 수 있다.
- **매칭 Lv.1**: 둘 다 Lv.1 이상이면 유리/불리/조건부/미확인 신호와 거친 성공 전망을 표시한다.
- **매칭 Lv.2**: 둘 다 Lv.2 이상이면 충돌/시너지 이유, 예상 성공률, 예상 정산 범위, 추가 베네핏 가능성을 표시한다.

### 2.1 매칭 신호 실시간 계산식 (Phase 1 이상 해금 시 적용)
플레이어가 용병을 슬롯에 등록하면 시스템은 해당 용병의 스탯, 장비, 자질(태그)을 기반으로 적합도를 동적 계산한다.

1. **유리 (Green Signal)**:
   - 조건: 용병의 특정 특성 태그가 미션의 최적화 매칭 태그(`#배수구_작업`, `#벤트_적응` 등)와 일치할 때.
   - 효과: 해당 노드 돌파 시 추가 확률 보정을 제공하거나 위험 페널티를 완전 무효화.
2. **불리 (Red Signal)**:
   - 조건: 용병의 기어 구성 중 무장 가시성이 기준치를 초과하거나, 미션 위협 요소와 상성이 맞지 않는 부정 특성 태그(예: `#소심함`, `#폐질환` 등)를 보유했을 때.
   - 효과: 감지 지수 급증, 혹은 특정 노드 진입 시 스탯 요구량 가산 페널티 부여.
3. **조건부 (Yellow Signal)**:
   - 조건: 용병의 특정 장비 장착 여부에 따라 생존 능력이 결정될 때. (예: `[인공 호흡 필터 마스크]`를 보유했으나 착용하지 않은 상태)
   - 효과: 기어 장착을 유도하는 경고 점멸 신호 발생.
4. **미확인 (Gray Signal)**:
   - 조건: 용병 분석 기관 또는 미션 분석 기관 중 한쪽의 레벨이 낮아 교차 해석을 완성할 수 없는 영역.
   - 효과: 해당 매칭 데이터 패널을 노이즈 잡음 및 물음표 상태로 출력한다. 필요한 분석 기관 또는 액티브 연산 슬롯을 안내한다.

### 2.2 실시간 출격 판정 (Deployment Verdict)
- **[가능 (Clear)]**: 필수 장비가 장착되어 있고, 경고 요인이 최소화된 상태.
- **[경고 (Warning)]**: 출격은 가능하지만, 필수 기어 누락 혹은 심각한 불리 자질로 인해 돌입 시 신체 훼손 및 의체 영구 마손 확률이 극도로 높음을 고지함.
- **[불가 (Locked)]**: 물리적 수사 게이트 통과를 위한 최소 바이오메트릭 인증 불가 등 절대적 필수 조건 미충족 상태.
