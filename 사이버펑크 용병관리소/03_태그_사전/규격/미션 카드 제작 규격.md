# 미션 카드 제작 규격

이 문서는 실제 미션 카드를 만들기 위한 1차 제작 규격이다.

최종 문체 확정 문서가 아니다. 목적은 현재까지 정리된 태그, 스탯, 장비, 정보 해금 규칙을 바탕으로 **플레이 테스트 가능한 미션 카드**를 만들 수 있게 하는 것이다.

## 1. 문서 목적

미션 카드는 플레이어에게 다음 질문을 던져야 한다.

```text
이 미션은 어떤 일을 시키는가?
어떤 용병이 맞을 것 같은가?
어떤 장비/임플란트/스테이션 준비가 필요할 것 같은가?
위험을 감수하고 보낼 만한가?
```

미션 카드는 정답지를 바로 주는 문서가 아니다.

Phase가 낮을수록 플레이어가 브리핑을 읽고 추론해야 하며, Phase가 높아질수록 시스템 요구가 명확히 드러난다.

핵심은 플레이어가 미션 브리핑의 줄글을 읽고 “어떤 성격의 의뢰인지”, “어떤 용병과 장비가 필요할지”를 스스로 판단하게 만드는 것이다.

## 2. 기본 원칙

- 미션은 단일 판정이 아니라 여러 노드로 진행되며, 난이도(별 개수)에 따라 노드 수가 스케일링된다. 기본값인 1성(쉬움)은 3개 노드이며, 난이도가 1성 상승할 때마다 관문 노드가 1개씩 추가된다. (예: 보통 2성 = 4개, 어려움 3성 = 5개 ...) 이를 통해 노드 증가에 따른 피로 누적, 의체 마모, 가시성 등 장기 매니지먼트를 유도한다.
- 정보 해금은 화면이 아예 교체되는 것이 아니라, **미션 분석 기관 레벨**에 따라 기존 미션 카드 하단에 분석 정보가 유기적으로 덧붙여지는 누적 노출형 구조를 취한다.
- 난이도, 계약 정산금(보상), 조기 철수 패널티와 같은 절대적인 계약의 비즈니스 상수는 Phase 0부터 상시 투명하게 노출된다.
- 태그는 단순하게 둔다.
- 미션이 태그, 스탯, 장비, 지역, 팩션, 상태를 복합적으로 읽는다.
- Phase 0에서는 필수/권장 조건을 직접 쓰지 않는다.
- Phase 1에서는 정답 대신 신호를 주며, 추천 스탯 범위(예: 프레임 [보통] 등)와 대략적인 권장 장비 카테고리 힌트를 제공한다.
- Phase 2에서는 시스템 판단에 필요한 정확한 조건(정량적 요구 스탯 수치 커트라인, 정밀 추천 장비 명칭, 추천 용병 특성 태그 명칭)을 해금한다. 단, 세부 노드 시뮬레이션 결과나 조커 카드 작동 메커니즘을 상세히 유출하여 플레이어의 탐색 재미를 해치지 않아야 한다.
- 이야기 핵심 비밀은 Phase 2에서도 숨길 수 있다.
- 미션 카드 자체는 특정 용병을 전제로 쓰지 않는다.
- 용병과 미션의 적합/부적합 해석은 미션 슬롯에 용병을 대입했을 때만 표시한다.
- 미션 카드의 Phase는 **미션 분석 기관 레벨**만 따른다. 용병 분석 기관 레벨이 높아도 미션 카드 자체의 정보가 대신 열리지는 않는다.
- 미션 슬롯 매칭의 표시 깊이는 **용병 분석 기관 레벨과 미션 분석 기관 레벨 중 낮은 쪽**을 따른다.
- 미션 브리핑은 태그 목록의 포장지가 아니라 플레이어 판단의 핵심 재료다.
- Phase 2에서도 브리핑을 대체하는 정답표만 남기지 않는다.
- 분석 정보는 브리핑을 읽고 세운 가설을 보조하거나 검증하는 역할이어야 한다.

## 2.1 미션 카드-용병 매칭 분리 원칙

미션 카드는 의뢰 자체의 정보다.

용병 이력서는 용병 자체의 정보다.

두 정보를 연결하는 해석은 별도 화면인 **미션 슬롯 매칭**에서 처리한다.

작성 규칙:

- 미션 카드에는 미션의 목표, 환경, 의뢰자, 요구 조건, 위험, 보상을 쓴다.
- 미션 카드 본문에는 특정 용병 이름이나 특정 용병의 장단점을 쓰지 않는다.
- Phase 1/2 매칭 신호는 용병을 슬롯에 올리고, 용병 분석 기관과 미션 분석 기관이 같은 단계 이상을 확보했을 때만 표시한다.
- "현재 슬롯 대입 용병 분석"은 미션 카드 기본 정보가 아니라 미션 슬롯 UI에 속한다.
- 미션 브리핑은 용병 이력서 내용을 역으로 설명하면 안 된다.
- 미션 브리핑은 플레이어가 요구 조건을 추론할 수 있을 만큼 구체적인 사건, 환경, 의뢰자 말투, 현장 단서를 포함한다.
- 단, 브리핑 본문이 `Frame 필요`, `Cypher 필요` 같은 정답 문장으로 바뀌면 안 된다.


모든 필드가 모든 Phase에서 보이는 것은 아니다.

## 5. 플레이어 노출 구조

플레이어에게 보이는 카드는 고립된 별개의 창이 아니며, 미션 분석 기관 레벨에 따라 정보가 아래로 유기적으로 덧붙여지는 누적 노출형 구조를 가진다.

### Phase 0 카드

목적:
- 플레이어가 텍스트를 읽고 필요한 조건을 스스로 추론한다.
- 직접 태그명, 필수 조건, 권장 조건, 위험 노드를 숨긴다.

보여줄 것:
- 의뢰명
- 난이도 (모든 페이즈 상시 고정 노출로 원본 추가 반영)
- 위치 (모든 페이즈 상시 고정 노출)
- 미션 타입 또는 의뢰 성격 (모든 페이즈 상시 고정 노출)
- 보상 및 조기 철수 패널티 (모든 페이즈 상시 고정 노출, 기권 위약금에서 용어 최신화)
- 브리핑 3~5문장

숨길 것:
- 요구 스탯 체급 및 구체적인 수치 조건
- 필수/권장 세부 장비 가이드
- 매칭 최적화 태그 명칭

작성 형식:

```text
## [Phase 0] 미션 카드

- 의뢰명:
- 난이도: ★★☆☆☆
- 위치:
- 의뢰 성격 (미션 타입):
- 보상 (계약 정산금):
- 조기 철수 패널티:

### 브리핑
3~5문장.
```

### Phase 1 카드

목적:
- 정답을 공개하지 않고, 맞는 방향과 위험 방향을 알려준다.
- 플레이어가 미션 분석 기관 업그레이드와 분석 슬롯 운용의 가치를 느끼게 한다.

보여줄 것:
- Phase 0 정보 일체 상시 누적 노출
- 출격 가능 / 경고 / 불가
- 매칭 신호 3~5개 (유리, 불리, 조건부, 미확인 포함)
- 대략적 위험도 또는 성공 전망
- 미확인 위험 존재
- 추천 스탯 체급 범위 (예: 프레임 [보통], 와이어 [보통] - 신규 병합 반영)
- 대략적인 권장 장비 카테고리 힌트 (예: "절연화 계열 기어 권장" - 신규 병합 반영)

숨길 것:
- 정확한 숨김 태그명
- 정밀한 장비 및 임플란트 세부 모델 규격 명칭
- 구체적인 정량 스탯 커트라인 숫자

작성 형식:

```text
## [Phase 1] 미션 카드
(Phase 0 기본 정보 일체 유지 및 하단 누적 덧붙임)

- 출격 판정: 가능 / 경고 / 불가
- 예상 판단: 낮음 / 보통 / 높음 같은 성공 전망

### 매칭 신호
- 유리: 
- 불리: 
- 조건부: 
- 미확인: 

### 1차 분석 가동: 추천 전술 세팅 (신규 병합)
- 권장 스탯 체급 범위:
  * 스탯명: [보통 / 높음 / 치명]
- 권장 장비 수급 단서:
  * 작전에 유용한 특정 장비 카테고리 명칭 힌트
```

### Phase 2 카드

목적:
- 시스템 판단에 필요한 조건을 명확히 보여준다.
- 후반 플레이어가 최적화 매니지먼트를 할 수 있게 한다.

보여줄 것:
- Phase 0, 1 정보 일체 상시 누적 노출
- 필수 조건 (정량 스탯 커트라인 및 정밀 장비 규격 모델명 명칭 추가 병합)
- 권장 조건 (최적화 추천 용병 특성 태그 명칭 추가 병합)
- 보너스 조건
- 요구 스탯
- 주요 위협 프로필 (세부 노드의 구체적 묘사 및 판정 수치를 숨기고 대표 위협 사건/피해 위주 노출)
- 현재 슬롯에 대입한 용병과의 충돌/시너지 (의뢰서-이력서 분리에 맞춰 슬롯에 대입 시에만 추가 연동)

숨길 것:
- 세부 노드의 개별 명칭 및 단계별 순찰 전개도
- 세부 노드별 내부 판정 수치 및 상세 계산식
- 비상 조커 카드의 정확한 내부 작동 연산식 및 확률
- 이야기 핵심 비밀 및 미확인 돌발 이벤트 결과

작성 형식:

```text
## [Phase 2] 미션 카드
(Phase 0, 1 정보 일체 유지 및 최하단 최종 누적 덧붙임)

### 시스템 요구
- 필수: (정밀 장비/임플란트 세부 규격 모델명 맵핑 포함)
- 권장: 
- 보너스:
- 요구 스탯: (구체적인 최소 요구 스탯 수치 커트라인 포함)

### 최적화 매칭 가이드 자질 (태그)
- 추천 태그 명칭: 해당 자질이 작전에 기여하는 우회로/리스크 대처 세부 힌트

### 위협 프로필 요약 (전체 노드의 스포일러는 숨기고 핵심 위험만 요약)
- 주요 위협 요소: (예: 고전압 전력망 방전 위험 [Frame, Wire 필요])
- 위기 발생 지점: (예: 2노드 돌파 실패 시 감전 위협 및 장비 파손 가능성 경고)
```

## 6. 필수/권장/보너스 조건 작성법

### 필수 조건

없으면 출격 불가 또는 강한 경고가 뜬다.

필수 조건 후보:

- 특정 환경에서 생존하기 위한 장비/임플란트.
- 기본 신체 파츠의 정상 작동.
- 보안 검문을 통과하기 위한 최소 기준.
- 미션 목표를 수행하기 위한 절대 조건.

작성 규칙:

- Phase 0에서는 직접 쓰지 않는다.
- Phase 1에서는 "출격 불가" 또는 "장비 부족" 정도로만 암시한다.
- Phase 2에서는 명확히 쓴다.

### 권장 조건

없어도 출격은 가능하지만 위험 노드가 증가한다.

권장 조건 후보:

- 낮은 가시성.
- 특정 스탯 기준.
- 특정 장비 성능.
- 지역 적응.
- 팩션 평판.
- 상태 안정성.

작성 규칙:

- 부족하면 부정 돌발 노드가 늘어난다.
- 초과 충족하면 긍정 돌발 또는 추가 보상이 생길 수 있다.
- Phase 0에서는 환경, 흔적, 주변 반응으로 암시한다.

### 보너스 조건

없어도 큰 불이익은 없지만 있으면 이득이 생긴다.

보너스 조건 후보:

- 소요 시간 단축.
- 추가 보상.
- 특수 로그.
- 후속 정보.
- 팩션 호감도.
- 장비 파손률 감소.

작성 규칙:

- 플레이어가 "과투자할지 말지" 고민하게 만들어야 한다.
- 보너스 조건을 필수처럼 쓰면 안 된다.

## 7. 노드 작성법

미션은 기본 관문 3~5개를 권장한다.

기본 관문 예:

```text
진입
목표 탐색
주요 작업
교전/회피
탈출
정산
```

노드마다 최소 하나의 판단 축을 둔다.

판단 축 예:

- 스탯.
- 장비.
- 임플란트.
- 가시성.
- 지역 위험.
- 팩션 반응.
- 현재 상태.
- 이야기 단서.

## 8. 위험 노드 작성법

위험 노드는 플레이어가 준비하지 못했을 때 생기는 긴장이다.

위험 노드 종류:

- 조건 부족 위험.
- 장비/임플란트 파손 위험.
- 신체/정신 상태 위험.
- 지역 환경 위험.
- 팩션 적대 위험.
- 이야기 태그 반응.

작성 규칙:

- 위험 노드는 미션의 핵심 판단과 연결되어야 한다.
- 아무 태그나 벌점으로 읽으면 안 된다.
- 위험 노드는 결과 보고서에서 설명 가능해야 한다.
- 신규 발생 상태와 파손은 결과 보고서에서 즉시 공개한다.

## 9. 조커 카드 작성법

조커 카드는 실패를 무마하는 준비 요소다.

조커 카드 후보:

- 높은 서브 스탯.
- 내장형 무기.
- 특정 임플란트 기능.
- 스테이션 1회 효과.
- 상태/성향 태그.
- 일부 이야기 태그.
- 팩션 평판.

작성 규칙:

- 조커 카드는 무조건 성공권이 아니다.
- 조커 카드 발동은 비용, 파손, 노출, 스트레스와 교환될 수 있다.
- 같은 조커 카드가 모든 미션을 해결하면 안 된다.

## 10. 브리핑 문장 작성 기준

Phase 0 브리핑은 정답지가 아니다.

좋은 브리핑은 다음을 한다.

- 미션의 분위기를 만든다.
- 필요한 조건의 방향을 암시한다.
- 위험의 흔적을 보여준다.
- 의뢰자나 지역의 성격을 보여준다.
- 플레이어가 판단할 여지를 남긴다.

나쁜 브리핑:

```text
이 미션은 호흡 필터와 낮은 가시성 장비가 필요하다.
협소 공간 공포가 있는 용병은 보내면 안 된다.
Frame 60 이상이면 밸브를 잠글 수 있다.
```

좋은 브리핑:

```text
먼저 들어간 작업자들이 남긴 산소통 안쪽은 누렇게 변색되어 있었다.
목표 지점으로 가는 정비 터널은 성인 한 명이 겨우 기어갈 폭이다.
임시 검문대는 장비 가방을 하나씩 열어보고 있었다.
```

## 11. 미션 타입별 작성 초점

### 전투

우선 읽는 것:

- 프레임.
- 무기 화력/명중/내구.
- 생존 임플란트.
- 장비 파손/보험.
- 전면 교전 태그.

카드에서 보여줄 긴장:

- 이길 수 있는가.
- 버틸 수 있는가.
- 장비가 얼마나 갈릴 것인가.
- 보험을 들 가치가 있는가.

### 잠입

우선 읽는 것:

- 사이퍼.
- 가시성.
- 은닉형 장비.
- 검문 대응.
- 흔적 제거.

카드에서 보여줄 긴장:

- 들키지 않고 들어갈 수 있는가.
- 들켰을 때 플랜 B가 있는가.
- 장비를 들고 들어갈 수 있는가.

### 지원

우선 읽는 것:

- 와이어.
- 수리/의료/배송/기술.
- 환경 대응 임플란트.
- 소요 시간 단축.

카드에서 보여줄 긴장:

- 일을 끝낼 기술이 있는가.
- 시간이 늦어지면 손해가 커지는가.
- 실패 시 위약금이나 신뢰 하락이 있는가.

### 기업

우선 읽는 것:

- 쿨.
- 사이퍼.
- 가시성.
- 팩션 평판.
- 인증/위장.
- 보안 검문.

카드에서 보여줄 긴장:

- 보상은 크지만 후폭풍이 큰가.
- 기업 보안망을 통과할 수 있는가.
- 실패 시 관계가 망가지는가.

### 비밀

우선 읽는 것:

- 펄스.
- 쿨.
- 미확인 위험.
- 이야기 태그.
- 돌발 노드 대응력.

카드에서 보여줄 긴장:

- 무엇이 진짜 목표인지 불명확한가.
- 성공 보상보다 실패 후폭풍이 큰가.
- 플레이어가 감으로 위험을 읽어야 하는가.

## 12. 결과 보고서 연결

미션 카드는 결과 보고서까지 생각하고 만들어야 한다.

결과 보고서에서 설명할 것:

- 어떤 조건이 충족되었는가.
- 어떤 조건이 부족했는가.
- 어떤 스탯/장비/태그가 위기를 만들었는가.
- 어떤 조커 카드가 개입했는가.
- 어떤 신규 상태가 생겼는가.
- 어떤 장비/임플란트가 파손되었는가.
- 어떤 보상/평판/후속 단서가 생겼는가.

신규 상태와 파손은 반드시 공개한다.

## 13. 작성 완료 체크리스트

미션 카드 하나를 만든 뒤 아래를 확인한다.

- Phase 0이 정답지처럼 보이지 않는가?
- Phase 1 신호가 5개 이하인가?
- Phase 2 핵심 요구가 8개 이하인가?
- 필수/권장/보너스 조건이 구분되는가?
- 기본 관문과 위험 노드가 구분되는가?
- 조커 카드 후보가 있는가?
- 스탯, 장비, 태그, 지역, 팩션 중 무엇을 읽는 미션인지 분명한가?
- 이야기 태그가 시스템 판단을 방해하지 않는가?
- 결과 보고서에서 성공/실패 이유를 설명할 수 있는가?
- 플레이어가 "보낼까 말까" 고민할 여지가 있는가?
- 미션 카드 본문이 특정 용병 이력서를 전제로 쓰이지 않았는가?
- 용병 적합/부적합 해석이 미션 슬롯 매칭 화면에만 있는가?

## 14. 현재 단계에서 하지 않는 것

- 실제 미션 대량 작성.
- 실제 태그 대량 작성.
- 수치 밸런스 확정.
- 최종 문체 확정.
- 모든 위험 노드의 확률 계산식 확정.

지금은 테스트 가능한 1차 미션 카드를 만들기 위한 규격을 잡는 단계다.
