# 상태 태그 후보군 1차

이 문서는 결과 보고서와 용병 이력서의 `현재 상태`에 반영될 수 있는 상태 태그 후보 범위를 정리한다.

중요:

- 이 문서는 최종 태그 사전이 아니다.
- 태그명, 효과, 지속 시간은 모두 1차 후보이며 더미다.
- 목적은 상태 태그와 스탯, 장비/임플란트 변화, 평판/노출 변화를 구분하는 것이다.

## 1. 상태 태그 정의

상태 태그는 용병의 현재 조건을 나타내는 시스템 태그다.

상태 태그가 답해야 하는 질문:

```text
이 용병을 다음 미션에 바로 보내도 되는가?
어떤 유형의 미션에서 유리하거나 불리해지는가?
휴식, 치료, 정비, 스테이션 조치가 필요한가?
이번 미션 결과가 용병의 다음 판단에 무엇을 남겼는가?
```

상태 태그는 스탯이 아니다.

따라서 `피로 +1단계`, `긴장도 2`, `Frame 저하`처럼 수치 증가나 스탯 감소처럼 쓰지 않는다.

## 2. 상태 태그가 아닌 것

아래 항목은 상태 태그로 우선 처리하지 않는다.

### 스탯값

금지:

```text
#낮은_Frame
#높은_Cypher
#Cool_부족
#Wire_경계값
```

이유:

- 스탯은 용병 기본 수치다.
- 스탯 부족/초과는 미션 슬롯 매칭이나 결과 보고서 Phase 2 판정에서 해석한다.

### 장비/임플란트 기능 변화

상태 태그로 만들지 않는 후보:

```text
#호흡기_자극
#필터오염
#연결구불량
#소형도구소모
```

처리 기준:

- 필터 오염, 출력 저하, 연결 불량, 도구 소모는 `장비/임플란트 변화`로 둔다.
- 그 결과 드러난 숨참, 기침, 떨림 같은 증상은 진행 로그나 증상 설명으로 둔다.
- 신체에 남은 후유증이 다음 파견 판단을 독립적으로 바꿀 때만 상태 태그 후보가 된다.

### 평판/수배/관계 변화

상태 태그로 만들지 않는 후보:

```text
#TheValve_호감
#QTransit_신뢰
#수배상승
```

처리 기준:

- 팩션 평판은 `평판/노출` 정산으로 둔다.
- 수배나 공식 노출은 결과 보고서의 `평판/노출` 영역에서 우선 처리한다.
- 특정 용병에게 붙은 신분 리스크가 다음 파견의 현재 상태로 작동할 때만 별도 상태 태그 후보가 될 수 있다.

## 3. 상태 태그 변화 방식

상태 태그는 가능한 한 추가/전환/완화/해소/소모로 표현한다.

기본 변화 방식:

```text
신규 부여: 없음 → #경계심
악화: #피로누적 → #심각한_피로누적
완화: #심각한_피로누적 → #피로누적
해소: #피로누적 → 제거
상위 전환: #양호한_컨디션 → #최상의_컨디션
하위 전환: #최상의_컨디션 → #양호한_컨디션
소모: #경계심 → 제거
병합: #피로누적 + 추가 피로 압력 → #심각한_피로누적
변질: #각성 → #흥분
```

작성 규칙:

- 같은 계열 상태가 중복되면 나란히 쌓기보다 전환을 우선 검토한다.
- 긍정 상태도 미션 결과로 생기거나 소모될 수 있다.
- 부정 상태도 모든 미션에서 즉시 실패를 뜻하지 않는다.
- 상태의 유불리는 미션 조건이 해석한다.

## 4. 후보군 분류

### 4.1 피로/회복 계열

역할:

- 장시간 작업, 연속 출격, 환경 부담, 회복 부족을 표현한다.
- 다음 미션의 집중도, 작업 지속력, 정밀 작업 안정성에 영향을 준다.

1차 후보:

```text
#피로누적
#심각한_피로누적
#회복중
#양호한_컨디션
```

사용 예:

```text
#피로누적 → #심각한_피로누적
원인: 장시간 체류와 작업 지연
효과: 다음 1회 장시간/정밀/환경 작업에서 집중도 저하 위험
완화: 휴식, 식사, 스테이션 회복
```

주의:

- `#피로`, `#피로도_상승`, `피로 +1단계`처럼 넓거나 수치처럼 보이는 표현은 피한다.
- 피로가 기존에 있으면 신규 부여보다 악화/상위 전환을 우선한다.

### 4.2 부상/신체 후유증 계열

역할:

- 미션 중 신체에 남은 손상이나 움직임 제한을 표현한다.
- 치료, 휴식, 보험, 의료 스테이션과 연결된다.

1차 후보:

```text
#경미한_부상
#타박상
#출혈
#신경통
```

사용 예:

```text
없음 → #경미한_부상
원인: 탈출 중 충돌
효과: 다음 1회 육체/추격/근접 대응 미션에서 불리
완화: 치료, 휴식
```

주의:

- 임플란트 파츠 파손은 이 계열이 아니라 장비/임플란트 변화다.
- 통증이나 기침은 증상 설명일 수 있다. 다음 파견 판단을 독립적으로 바꿀 때만 상태 태그로 올린다.

### 4.3 정신/압박 계열

역할:

- 협박, 총격, 과잉 경보, 실패 경험, 민간인 피해 같은 정신적 압박을 표현한다.
- `Cool`, 협상, 제한 교전, 민간인 관리 미션과 연결될 수 있다.

1차 후보:

```text
#긴장
#동요
#각성
#흥분
#침착함
#편집증
```

사용 예:

```text
없음 → #긴장
원인: 민간인 피해가 난 현장에서 장시간 대치
효과: 다음 1회 협상/검문/제한 교전 미션에서 과잉 대응 위험
완화: 휴식, 상담, 유흥, 스테이션 안정화
```

주의:

- `#침착함` 같은 긍정 상태도 모든 미션에서 좋은 것은 아니다. 긴급 대응 미션에서는 반응이 늦게 읽힐 수 있다.
- `#편집증`은 단순 경계보다 강한 부정 상태다. 반복 위협, 배신, 감시 공포 같은 정신 후유증에 쓴다.
- 성격 태그와 섞지 않는다. 성격은 상시 정보, 상태는 현재 조건이다.

### 4.4 경계/감지 계열

역할:

- 이상 동선, 침입 노드, 오인 경보, 매복 위험을 조기에 읽는 현재 집중 상태를 표현한다.
- 경비, 감시, 검문, 잠입 방어 미션과 연결된다.

1차 후보:

```text
#경계심
#경계태세
#예리함
```

사용 예:

```text
없음 → #경계심
원인: 야간 경비 중 오인 교전 없이 이상 동선을 계속 확인
효과: 다음 1회 경비/감시/검문 미션에서 침입 노드 또는 이상 동선 조기 감지에 유리
소모: 다음 출격 후 제거
```

주의:

- `#근무리듬_안정`처럼 효과가 바로 읽히지 않는 이름은 피한다.
- 감각 임플란트의 기능 향상/저하는 상태 태그가 아니라 장비/임플란트 변화다.
- `#예리함`은 일시적으로 감이 날카로운 상태일 때만 쓴다. 상시 능력이나 성격처럼 쓰지 않는다.
- `#경계태세`는 가벼운 경계 보정보다 강하고, `#편집증`보다는 덜 병적인 상태로 둔다.

### 4.5 환경 노출 계열

역할:

- 독성, 열, 냉기, 방사, 오염 같은 환경이 용병 몸에 남긴 현재 부담을 표현한다.
- 환경 미션, 치료, 보호 장비 준비와 연결된다.

1차 후보:

```text
#독성노출
#열피로
#저체온
#잔류오염
```

사용 예:

```text
없음 → #독성노출
원인: 보호 장비가 있었지만 오염 구역 체류 시간이 길어짐
효과: 다음 1회 환경/장시간 작업에서 위험 증가
완화: 해독, 휴식, 의료 스테이션
```

주의:

- 보호 장비의 필터 오염은 장비 변화다.
- 용병 몸에 남은 노출 후유증만 상태 태그 후보로 둔다.

### 4.6 긍정/조건부 컨디션 계열

역할:

- 좋은 결과, 안정적 수행, 충분한 회복으로 생긴 일시적 이점을 표현한다.
- 다음 1회 또는 특정 조건에서만 유리하게 작동한다.

1차 후보:

```text
#양호한_컨디션
#최상의_컨디션
#침착함
#경계심
```

사용 예:

```text
없음 → #양호한_컨디션
원인: 충분한 휴식 후 무사고 임무 완료
효과: 다음 1회 일반/정밀 작업에서 안정성 상승
소모: 다음 출격 후 제거
```

주의:

- 긍정 상태를 너무 많이 만들면 장비/스테이션 준비의 의미가 약해진다.
- 긍정 상태는 대부분 지속형보다 소모형을 우선 검토한다.

## 5. 이름 작성 기준

상태 태그명은 짧고 직관적이어야 한다.

상태 태그는 UI에 직접 들어가는 단어다.

따라서 설명문, 판정 과정, 행동 결과, 시스템 계산처럼 보이는 표현을 쓰지 않는다.

기본형은 캐릭터의 현재 상태를 보여주는 짧은 상태명이다.

한 단어 명사만 고집하지 않는다.

다른 게임 UI에서 상태 이상이나 버프 이름처럼 자연스럽게 읽히는 짧은 복합어는 허용한다.

조사나 수식어도 짧은 상태명으로 자연스럽다면 허용한다.

좋은 이름 기준:

- 플레이어가 이름만 보고 대략의 효과를 떠올릴 수 있다.
- 다음 선택과 연결된다.
- 스탯값, 장비 파츠, 팩션 평판을 대신하지 않는다.
- 너무 문학적이거나 추상적이지 않다.
- `유지`, `증폭`, `판정`, `보정`, `수치`, `성공률`처럼 시스템 계산이나 효과 설명처럼 보이는 꼬리를 남발하지 않는다.
- `경미한 부상`, `양호한 컨디션`, `최상의 컨디션`, `독성 노출`처럼 UI 상태명으로 자연스러운 표현은 허용한다.

좋은 방향:

```text
#경계심
#피로누적
#심각한_피로누적
#긴장
#예리함
#경계태세
#독성노출
#침착함
#경미한_부상
#양호한_컨디션
#최상의_컨디션
```

나쁜 방향:

```text
#근무리듬_안정
#현장감각의_여운
#호흡기_자극
#Frame_저하
#피로도_상승
#침착유지
#의심증폭
#감각예민
```

## 6. 결과 보고서 표시 양식

상태 태그는 결과 보고서에서 이름만 표시하지 않는다.

기본 표시:

```text
상태 태그 변화: 없음 → #경계심
변화 유형: 긍정/조건부 상태 태그 신규 부여
원인: 야간 경비 중 오인 교전 없이 이상 동선을 계속 확인
효과: 다음 1회 경비/감시/검문 미션에서 침입 노드 또는 이상 동선 조기 감지에 유리
해소/소모: 다음 출격 후 제거
다음 반영: 용병 이력서 현재 상태에 즉시 표시
```

작성 규칙:

- 상태 태그 변화는 Phase와 무관하게 표시한다.
- Phase가 낮아도 플레이어가 만든 결과는 숨기지 않는다.
- 원인 설명은 진행 로그와 연결되어야 한다.
- 효과 설명은 다음 선택에 도움이 되어야 한다.

## 7. 1차 확정 후보와 보류 후보

### 1차 확정 후보

현재 샘플에서 이미 검증된 후보:

```text
#피로누적
#심각한_피로누적
#경계심
```

확정 이유:

- 결과 보고서 샘플에서 실제 정산 위치가 검증되었다.
- 효과가 다음 미션 판단으로 이어진다.
- 장비/임플란트 변화와 구분된다.

### 검토 후보

추가 샘플에서 검증할 후보:

```text
#긴장
#동요
#각성
#흥분
#편집증
#경계태세
#예리함
#경미한_부상
#독성노출
#양호한_컨디션
#최상의_컨디션
```

검토 이유:

- 상태 태그로 의미는 있으나 아직 미션 샘플에서 반복 검증되지 않았다.
- 긍정 상태의 유지/소모 규칙이 아직 필요하다.
- 환경 노출과 장비 보호 실패의 경계를 더 테스트해야 한다.

### 보류 또는 금지 후보

현재 기준에서 상태 태그로 올리지 않는 후보:

```text
#호흡기_자극
#필터오염
#소형도구소모
#근무리듬_안정
#낮은_Frame
#피로도_상승
```

보류/금지 이유:

- 장비/임플란트 변화와 섞인다.
- 스탯값을 태그화한다.
- 효과가 직관적으로 읽히지 않는다.
- 수치 게이지처럼 보인다.

## 8. 다음 검증 과제

다음 작업에서는 아래를 정해야 한다.

1. 장비/임플란트 변화 후보군 1차.
2. 상태 태그와 장비 변화가 동시에 발생할 때의 표시 순서.
3. 긍정 상태 태그의 유지/소모 규칙.
4. 환경 노출 상태와 보호 장비 파손의 경계.
5. 상태 태그가 치료비, 휴식, 스테이션 기능과 연결되는 방식.
