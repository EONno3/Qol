# 장비 임플란트 변화 통합 검토본

이 문서는 장비 변화와 임플란트 변화를 한 파일에 함께 검토했던 통합 초안이다.

중요:

- 이 문서는 실제 사전으로 우선 사용하지 않는다.
- 실제 사용 기준은 `장비 변화 사전 1차.md`와 `임플란트 변화 사전 1차.md`를 우선한다.
- 이 문서는 두 사전의 경계와 누락을 비교하는 검토본으로만 둔다.
- 수리비, 교체비, 내구도 수치는 아직 확정하지 않는다.
- 장비/임플란트 변화는 용병 `상태 태그`와 분리한다.
- 장비/임플란트 문제로 나타난 신체 반응은 결과 로그에서 묘사할 수 있다.
- 다만 정산 항목은 신체 반응 묘사가 아니라 실제 원인인 장비/임플란트 변화로 기록한다.

## 1. 작성 기준

장비/임플란트 변화는 장착물, 소모품, 파츠, 필터, 연결구, 무기, 보호구에 생긴 결과다.

기본 필드:

```text
변화명
대상
성격
대표 효과
주요 변화 관계
복구/정비 조건
상태 태그와 구분
결과 보고서 표기
오용 금지
```

표기 원칙:

- 범용 변화명은 일반 명사로 둔다.
- 특정 장비나 파츠에 고정되는 변화는 장비/임플란트 상태 식별자로 쓸 수 있다.
- `Visibility` 변화는 태그가 아니라 장비 속성 변화로 둔다.

예:

```text
정밀 손가락 툴: 폐수 오염 / 세척 필요
호흡 보조 장비: #호흡보조_불안정 → #호흡보조_기능저하
소형 비살상 도구: 1회 소모
외장형 중화기: 내구도 감소 / 정비 필요
```

금지:

```text
#호흡기_자극
#필터오염을 용병 상태 태그로 표시
#낮은_Visibility
#Frame_저하
```

## 2. 오염/세척 계열

### 오염

- 대상: 외장 장비, 손 도구, 보호구, 임플란트 외부 접속부.
- 성격: 정비 필요 변화.
- 대표 효과: 다음 사용 시 신뢰도 저하, 정비비 발생 후보.
- 주요 변화 관계:
  - 정상 → 오염
  - 오염 → 기능 저하
  - 오염 → 정상
- 복구/정비 조건: 세척, 부품 점검, 스테이션 정비.
- 상태 태그와 구분: 용병 몸에 남은 독성은 `#독성노출`, 장비에 묻은 폐수/먼지는 오염.
- 결과 보고서 표기: `정밀 손가락 툴: 폐수 오염. 세척 필요.`
- 오용 금지: 신체 후유증이나 환경 노출 상태로 뭉개지 않는다.

### 잔류 이물질

- 대상: 필터, 관절부, 장갑판 틈, 센서 포트.
- 성격: 경미한 정비 필요 변화.
- 대표 효과: 다음 1회 장시간 사용 시 기능 저하 위험.
- 주요 변화 관계:
  - 정상 → 잔류 이물질
  - 잔류 이물질 → 오염
  - 잔류 이물질 → 정상
- 복구/정비 조건: 간이 세척, 소모품 교체.
- 상태 태그와 구분: 이물질은 장비 문제다. 용병 신체 상태가 아니다.
- 결과 보고서 표기: `센서 포트: 이물질 잔류 / 간이 세척 필요.`
- 오용 금지: 모든 오염을 같은 강도로 처리하지 않는다.

## 3. 소모/충전 계열

### 1회 소모

- 대상: 소형 비살상 도구, 일회용 드론, 약제 카트리지, 응급 키트.
- 성격: 소모품 변화.
- 대표 효과: 재보급 전까지 같은 장비 효과 사용 불가.
- 주요 변화 관계:
  - 보유 → 1회 소모
  - 1회 소모 → 재보급 완료
- 복구/정비 조건: 재구매, 충전, 보급.
- 상태 태그와 구분: 도구 소모는 용병 상태가 아니다.
- 결과 보고서 표기: `소형 비살상 도구: 1회 소모.`
- 오용 금지: 미획득 보너스나 손실과 섞지 않는다. 실제 보유 자원이 줄었으므로 손실/소모다.

### 충전 부족

- 대상: 전자 장비, 방어막, 드론, 에너지 무기, 내장형 보조 장치.
- 성격: 재충전 필요 변화.
- 대표 효과: 다음 출격 전 충전하지 않으면 일부 기능 제한.
- 주요 변화 관계:
  - 정상 → 충전 부족
  - 충전 부족 → 방전
  - 충전 부족 → 정상
- 복구/정비 조건: 충전, 배터리 교체, 스테이션 전력 사용.
- 상태 태그와 구분: 용병 피로가 아니라 장비 에너지 문제다.
- 결과 보고서 표기: `보조 방어막: 충전 부족 / 재충전 필요.`
- 오용 금지: `#피로누적` 같은 상태 태그로 대체하지 않는다.

### 방전

- 대상: 전자 장비, 방어막, 드론, 에너지 무기.
- 성격: 기능 제한 변화.
- 대표 효과: 재충전 전까지 장비 기능 사용 불가.
- 주요 변화 관계:
  - 충전 부족 → 방전
  - 방전 → 충전 부족
  - 방전 → 정상
- 복구/정비 조건: 충전, 배터리 교체.
- 상태 태그와 구분: 용병 상태가 아니라 장비 작동 상태다.
- 결과 보고서 표기: `정찰 드론: 방전 / 다음 출격 전 충전 필요.`
- 오용 금지: 파손과 혼동하지 않는다. 방전은 복구 비용과 시간이 다르다.

## 4. 기능 저하 계열

### 기능 불안정

- 대상: 임플란트, 센서, 보조 장비, 연결형 무기.
- 성격: 경고 단계 변화.
- 대표 효과: 다음 사용 시 기능 저하나 돌발 노드 위험.
- 주요 변화 관계:
  - 정상 → 기능 불안정
  - 기능 불안정 → 기능 저하
  - 기능 불안정 → 정상
- 복구/정비 조건: 점검, 필터 교체, 접속부 보정.
- 상태 태그와 구분: 결과 로그에는 몸의 반응을 묘사할 수 있다. 다만 원인이 장비 출력 흔들림이라면 정산 항목은 장비 기능 불안정이다.
- 결과 보고서 표기: `호흡 보조 장비: 출력 불안정 / 점검 필요.`
- 오용 금지: 곧바로 부상 태그로 바꾸지 않는다.

### 기능 저하

- 대상: 임플란트, 보조 장비, 무기, 센서.
- 성격: 부정 장비 변화.
- 대표 효과: 해당 장비가 읽히는 미션에서 신뢰도 하락 또는 추가 위험.
- 주요 변화 관계:
  - 기능 불안정 → 기능 저하
  - 기능 저하 → 기능 불안정
  - 기능 저하 → 정상
  - 기능 저하 → 파손
- 복구/정비 조건: 정비, 부품 교체, 전문 수리.
- 상태 태그와 구분: 기능 저하는 장비/임플란트 변화다. 신체 증상은 별도 설명한다.
- 결과 보고서 표기: `#호흡보조_불안정 → #호흡보조_기능저하`
- 오용 금지: `#호흡기_자극` 같은 상태 태그로 대체하지 않는다.

### 출력 저하

- 대상: 호흡 보조, 순환계 보조, 신경계 보조, 외장형 보조 장치.
- 성격: 기능 저하의 세부 변화.
- 대표 효과: 장시간/환경/전투 지속 미션에서 장비 보조 신뢰도 하락.
- 주요 변화 관계:
  - 정상 → 출력 저하
  - 출력 저하 → 기능 저하
  - 출력 저하 → 정상
- 복구/정비 조건: 출력 보정, 필터 교체, 부품 점검.
- 상태 태그와 구분: 결과 로그에는 호흡이 흔들린 장면을 쓸 수 있다. 다만 정산 항목은 신체 반응이 아니라 장비 출력 저하다.
- 결과 보고서 표기: `호흡 보조 장비: 출력 저하 / 필터 교체 필요.`
- 오용 금지: 피로 상태와 자동 병합하지 않는다.

## 5. 파손/손실 계열

### 경미한 파손

- 대상: 외장 장비, 무기, 보호구, 임플란트 외부 모듈.
- 성격: 수리 필요 변화.
- 대표 효과: 다음 사용 시 일부 기능 불안정, 수리비 발생.
- 주요 변화 관계:
  - 정상 → 경미한 파손
  - 경미한 파손 → 파손
  - 경미한 파손 → 정상
- 복구/정비 조건: 수리, 부품 교체.
- 상태 태그와 구분: 용병 부상과 별도다.
- 결과 보고서 표기: `외장형 방어판: 경미한 파손 / 수리비 발생.`
- 오용 금지: 장비 파손을 `#경미한_부상`으로 표시하지 않는다.

### 파손

- 대상: 무기, 임플란트 파츠, 보호구, 이동 파츠.
- 성격: 강한 부정 장비 변화.
- 대표 효과: 수리 전까지 장비 기능 제한 또는 출격 제한 후보.
- 주요 변화 관계:
  - 기능 저하 → 파손
  - 경미한 파손 → 파손
  - 파손 → 기능 저하
  - 파손 → 정상
- 복구/정비 조건: 전문 수리, 부품 교체, 보험 처리.
- 상태 태그와 구분: 파손 부위가 용병 몸에 붙어 있어도 장비/임플란트 변화로 먼저 표시한다.
- 결과 보고서 표기: `하완 임플란트: 파손 / 수리 전 출격 제한 후보.`
- 오용 금지: 출격 불가 판정과 자동 동일시하지 않는다. 파츠별 강제 조건을 따진다.

### 영구 손실

- 대상: 회수 실패 장비, 파괴된 소모품, 완전 손상 파츠.
- 성격: 손실.
- 대표 효과: 재구매 또는 재시술 필요.
- 주요 변화 관계:
  - 파손 → 영구 손실
  - 영구 손실 → 재구매/재시술 완료
- 복구/정비 조건: 새 장비 구매, 임플란트 재시술, 보험 보상.
- 상태 태그와 구분: 장비가 사라진 것이지 용병 현재 상태가 아니다.
- 결과 보고서 표기: `정찰 드론: 회수 실패 / 영구 손실.`
- 오용 금지: 추가 베네핏 미획득과 혼동하지 않는다. 이미 보유한 장비를 잃었으면 손실이다.

## 6. 연결/동기화 계열

### 연결 불량

- 대상: 내장형 무기, 임플란트, 외부 장비 접속부, 손목 연결구.
- 성격: 기능 불안정 변화.
- 대표 효과: 장비 시너지 실패, 조작 지연, 돌발 노드 위험.
- 주요 변화 관계:
  - 정상 → 연결 불량
  - 연결 불량 → 기능 저하
  - 연결 불량 → 정상
- 복구/정비 조건: 접속부 조정, 펌웨어 재동기화, 정비.
- 상태 태그와 구분: 결과 로그에는 조작이 흔들린 장면을 쓸 수 있다. 다만 원인이 접속 문제라면 정산 항목은 장비 변화다.
- 결과 보고서 표기: `손목 연결구: 연결 불량 / 재동기화 필요.`
- 오용 금지: `#신경통`과 섞지 않는다. 신경통은 신체 후유증이다.

### 동기화 오류

- 대상: 스마트 무기, 시각/청각 보조, 드론, 사이버덱.
- 성격: 시스템/펌웨어 변화.
- 대표 효과: 시너지 비활성, 오인식별, 지연 반응 위험.
- 주요 변화 관계:
  - 정상 → 동기화 오류
  - 동기화 오류 → 기능 저하
  - 동기화 오류 → 정상
- 복구/정비 조건: 펌웨어 점검, 재동기화, 스테이션 장비 분석.
- 상태 태그와 구분: 오인 경보는 사건 로그, 원인은 장비 동기화 오류일 수 있다.
- 결과 보고서 표기: `보안 드론 로그: 동기화 오류 의심.`
- 오용 금지: 후속 단서와 장비 변화가 같지 않다. 회수한 로그는 후속 단서, 장비 상태는 별도다.

## 7. 필터/소모 부품 계열

### 필터 오염

- 대상: 호흡 보조, 환경 보호구, 후각/구강 임플란트 보조 필터.
- 성격: 소모 부품 변화.
- 대표 효과: 환경 미션에서 보호 신뢰도 하락.
- 주요 변화 관계:
  - 정상 → 필터 오염
  - 필터 오염 → 기능 저하
  - 필터 오염 → 정상
- 복구/정비 조건: 필터 교체, 세척.
- 상태 태그와 구분: 용병이 독성에 노출됐으면 `#독성노출`, 필터가 더러워졌으면 필터 오염.
- 결과 보고서 표기: `호흡 보조 필터: 오염 / 교체 필요.`
- 오용 금지: `#필터오염`을 용병 상태 태그로 올리지 않는다.

### 부품 마모

- 대상: 관절, 손가락 툴, 외장 무기, 이동 파츠.
- 성격: 누적 정비 변화.
- 대표 효과: 반복 사용 시 기능 불안정 또는 파손 위험 증가.
- 주요 변화 관계:
  - 정상 → 부품 마모
  - 부품 마모 → 기능 불안정
  - 부품 마모 → 정상
- 복구/정비 조건: 부품 교체, 정비.
- 상태 태그와 구분: 용병 피로와 별도다.
- 결과 보고서 표기: `대퇴부 지지 관절: 부품 마모 / 정비 권장.`
- 오용 금지: 단일 미션 손상과 장기 내구도 시스템을 섞지 않는다.

## 8. Visibility 속성 변화

### Visibility 상승

- 대상: 외장형 무기, 장비 조합, 노출된 임플란트, 임시 수리 흔적.
- 성격: 장비 속성 변화.
- 대표 효과: 검문/잠입/기업/보안 미션에서 위험 증가 후보.
- 주요 변화 관계:
  - 기본 Visibility → 일시 Visibility 상승
  - 일시 Visibility 상승 → 기본 Visibility
- 복구/정비 조건: 위장 처리, 장비 교체, 외장 커버 수리.
- 상태 태그와 구분: `Visibility`는 태그가 아니라 장비 속성/수치다.
- 결과 보고서 표기: `장비 노출도: 임시 수리 흔적으로 상승 가능.`
- 오용 금지: `#낮은_Visibility`, `#높은_Visibility` 같은 태그를 만들지 않는다.

### Visibility 안정

- 대상: 위장 장비, 은닉형 무기, 내장형 장비.
- 성격: 장비 속성 정상 유지.
- 대표 효과: 보안 드론/검문 위협 점수 상승 없음.
- 주요 변화 관계:
  - Visibility 상승 → Visibility 안정
  - Visibility 안정 → Visibility 상승
- 복구/정비 조건: 위장 유지, 장비 은닉 상태 유지.
- 상태 태그와 구분: 용병 상태가 아니라 장비 노출 속성이다.
- 결과 보고서 표기: `장비 노출 관련 기록: 특이 상승 없음.`
- 오용 금지: 긍정 상태 태그로 만들지 않는다.

## 9. 보험/정비 정산 계열

### 정비 필요

- 대상: 모든 장비/임플란트.
- 성격: 비용 후보 변화.
- 대표 효과: 정비 전까지 기능 불안정 또는 비용 발생.
- 주요 변화 관계:
  - 오염/마모/기능 저하/파손 → 정비 필요
  - 정비 필요 → 정상
- 복구/정비 조건: 크레딧 지불, 스테이션 정비, 부품 교체.
- 상태 태그와 구분: 정비 필요는 장비 상태다.
- 결과 보고서 표기: `정비비: -180 크레딧.`
- 오용 금지: 치료비와 혼동하지 않는다.

### 보험 처리

- 대상: 보험 적용 장비, 임플란트, 외장형 무기.
- 성격: 비용 완화 정산.
- 대표 효과: 수리비 감소, 영구 손실 복구 후보.
- 주요 변화 관계:
  - 파손 → 보험 처리
  - 영구 손실 → 보험 보상 후보
- 복구/정비 조건: 사전 보험 가입, 계약 조건 충족.
- 상태 태그와 구분: 보험은 용병 상태가 아니라 정산 조건이다.
- 결과 보고서 표기: `장비 보험 적용: 수리비 일부 감소.`
- 오용 금지: 보험 처리를 보상으로 과장하지 않는다. 손실 완화다.

## 10. 1차 보류 변화

아래 후보는 장비/임플란트 변화로 바로 채택하지 않는다.

```text
#호흡기_자극
#피로누적
#경미한_부상
#낮은_Frame
#높은_Visibility
```

보류 이유:

- 신체 증상 또는 상태 태그다.
- 스탯값을 태그화한다.
- `Visibility` 속성을 태그화한다.
- 장비 문제의 결과 증상과 장비 변화 원인을 섞는다.
