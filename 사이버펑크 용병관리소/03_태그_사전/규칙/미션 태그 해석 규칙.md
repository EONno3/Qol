# 미션 태그 해석 규칙

이 문서는 미션이 태그를 어떻게 읽는지 정하는 기준 문서다.

실제 미션이나 실제 태그 목록을 작성하지 않는다. 미션 타입, 노드, 필수/권장/보너스 조건이 태그를 해석하는 방식을 정한다.

## 1. 핵심 원칙

태그는 고정 효과가 아니다.

같은 태그라도 미션에 따라 다르게 읽힌다.

```text
태그는 단순하게 둔다.
미션이 복잡하게 읽는다.
```

따라서 태그 자체에 "항상 성공률 +N" 같은 효과를 먼저 넣지 않는다.

## 2. 기본 해석값

미션은 태그를 네 방식 중 하나로 읽는다.

```text
긍정: 현재 미션에서 유리하게 작동.
부정: 현재 미션에서 위험으로 작동.
무시: 현재 미션에서는 중요하지 않음.
특수: 특정 노드/조건/이야기에서만 반응.
```

예:

- 전투 미션에서 외형 태그는 무시될 수 있다.
- 잠입 미션에서 외형/가시성 태그는 핵심 위험이 될 수 있다.
- 상층 사교 미션에서 출신/말투/의전 관련 태그는 특수 반응을 만들 수 있다.

## 3. 미션 판정 흐름

미션은 단일 판정이 아니다.

기본 흐름:

```text
1. 미션 타입 확인
2. 필수 조건 확인
3. 권장 조건 확인
4. 주요 스탯과 장비 세팅 확인
5. 기본 관문 생성
6. 관문 돌파 판정
7. 실패/조건 부족/조건 과잉에 따라 돌발 노드 발생
8. 조커 카드 개입
9. 결과 보고서와 정산
```

## 4. 조건 단계

미션 조건은 세 단계로 나눈다.

### 필수 조건

없으면 출격 불가 또는 강제 경고가 뜬다.

예:

- 특정 환경 생존 장비.
- 특정 신체 파츠 정상 작동.
- 특정 보안 기준 통과.

### 권장 조건

없어도 출격은 가능하지만 위험 노드가 증가한다.

권장 조건을 넘치게 충족하면 긍정 돌발이나 추가 보상이 생길 수 있다.

### 보너스 조건

없어도 불이익은 적다.

있으면 소요 시간 단축, 추가 보상, 특수 로그, 후속 정보 획득이 가능하다.

## 5. 노드 종류

### 기본 관문

미션이 원래 가지는 진행 단계다.

예:

```text
진입
목표 탐색
주요 작업
탈출
정산
```

기본 관문은 주로 스탯, 장비, 시스템 태그를 읽는다.

### 돌발 노드

조건 부족, 상태, 위험, 이야기 태그 때문에 기본 관문을 대체하거나 끼어드는 노드다.

돌발 노드는 무조건 나쁜 것만은 아니다.

- 부족 조건: 부정 돌발.
- 권장 초과 충족: 긍정 돌발.
- 이야기 태그 반응: 특수 돌발.

### 조커 카드

관문 실패나 돌발 위험을 무마하는 준비 요소다.

조커 카드 후보:

- 서브 스탯.
- 내장형 무기.
- 임플란트 특수 기능.
- 스테이션 1회 효과.
- 상태/성향 태그.
- 일부 이야기 태그.

## 6. 미션 타입별 우선 해석

### 전투

우선 읽는 것:

- 프레임.
- 무기 화력/명중/내구.
- 생존 관련 임플란트.
- 전면 교전 태그.
- 장비 파손/보험.

덜 읽는 것:

- 의전.
- 미세한 외형.
- 상층 사교성.

단, 특정 전투 미션이 공개 경기, 상층 결투, 암살전이면 외형/평판도 읽을 수 있다.

### 잠입

우선 읽는 것:

- 사이퍼.
- 가시성.
- 은닉형 장비.
- 검문 대응.
- 흔적 제거.
- 조용한 무기.

덜 읽는 것:

- 순수 화력.
- 고가시성 외장 장비.

단, 발각 후 전투를 전제로 한 잠입은 플랜 B 장비를 읽는다.

### 지원

우선 읽는 것:

- 와이어.
- 장비 조작.
- 수리/의료/배송/기술 태그.
- 환경 대응 임플란트.
- 소요 시간 단축 요소.

덜 읽는 것:

- 순수 살상력.
- 의전 외형.

지원 미션은 실패 시 위약금, 장비 손상, 신뢰도 하락과 연결하기 좋다.

### 공작

우선 읽는 것:

- 쿨.
- 사이퍼.
- 가시성.
- 팩션 평판.
- 장비 인증/위장.
- 보안 검문.

공작 미션은 보상은 크지만 실패 후폭풍이 커야 한다.

### 비밀

우선 읽는 것:

- 펄스.
- 쿨.
- 숨은 정보.
- 이야기 태그.
- 미확인 위험.
- 돌발 노드 대응력.

비밀 미션은 요구가 불명확할 수 있다. Phase가 낮을수록 플레이어 추론 비중이 커진다.

## 7. 태그 조합 해석

미션은 단일 태그보다 조합을 읽는다.

조합 예시 구조:

```text
스탯 + 장비 + 태그 + 지역 + 팩션 + 현재 상태
```

조합 해석 원칙:

- 하나의 강점 태그가 모든 위험을 지우지 않는다.
- 강점이 있어도 상태가 나쁘면 돌발 노드가 생긴다.
- 장비가 좋아도 가시성이 높으면 잠입에서 불리할 수 있다.
- 이야기 태그는 보통 직접 보정이 아니라 특수 반응을 만든다.

## 8. 정보 노출과 미션 해석

미션 해석 결과는 Phase에 따라 다르게 보여준다.

### Phase 0

- 필수/권장 조건 직접 공개 금지.
- 태그명 직접 공개 금지.
- 브리핑과 흔적으로 추론하게 한다.

### Phase 1

- 유리/불리/조건부/미확인 신호 제공.
- 정확한 태그명과 계산식은 숨긴다.
- 출격 가능/경고/불가 정도는 보여줄 수 있다.

### Phase 2

- 필수 조건, 권장 조건, 위험 노드, 충돌 이유 공개.
- 조커 카드 후보 표시 가능.
- 이야기 핵심 비밀은 여전히 숨길 수 있다.

## 9. 결과 보고서

결과 보고서는 판정 결과를 설명한다.

반드시 연결할 것:

- 어떤 조건이 충족/미충족되었는가.
- 어떤 태그/장비/스탯이 위기를 만들었는가.
- 어떤 조커 카드가 개입했는가.
- 어떤 신규 상태, 파손, 보상, 평판 변화가 생겼는가.

신규 상태와 파손은 즉시 공개한다.

## 10. 검토 질문

미션을 만들 때 아래를 확인한다.

```text
1. 이 미션의 주요 판단은 스탯인가, 장비인가, 태그인가, 이야기인가?
2. 필수/권장/보너스 조건이 나뉘는가?
3. 기본 관문과 돌발 노드가 구분되는가?
4. 조커 카드 후보가 있는가?
5. Phase 0에서 너무 정답지를 주고 있지 않은가?
6. Phase 2에서 필요한 시스템 정보가 충분히 공개되는가?
```
