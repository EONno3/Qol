# 이력서 및 미션 카드 압축 규칙

이 문서는 용병 이력서와 미션 카드가 너무 길어져 플레이어가 읽기 어려워지는 것을 막기 위한 기준 문서다.

현재 단계에서는 실제 문구와 말투를 확정하지 않는다. 먼저 **화면에 얼마나 보여줄지**와 **어떤 정보는 접어둘지**를 정한다.

## 1. 핵심 원칙

- 플레이어는 첫 화면에서 10초 안에 핵심 판단을 할 수 있어야 한다.
- 텍스트는 분위기를 만들지만, 판단 정보는 압축되어야 한다.
- Phase 0은 추론용 텍스트, Phase 1은 요약 신호, Phase 2는 상세 분석이다.
- 모든 정보를 한 번에 펼치지 않는다.
- 이야기 태그는 시스템 태그보다 덜 자주, 더 조심스럽게 노출한다.

## 1.1 정보 노출 강도 규칙과의 관계

이 문서는 **표시량과 화면 압축**을 정한다.

단, 어떤 정보를 숨기거나 공개할지는 `정보 노출 강도 규칙.md`를 우선한다.

우선순위:

```text
1. 정보 노출 강도 규칙
2. 이력서 및 미션 카드 압축 규칙
3. 실제 UI 문구/말투
```

즉, 이 문서의 표시 수 제한은 Phase별 공개 원칙을 보조하는 규칙이다.

예:

- Phase 0은 짧게 쓰되, 직접 태그명이나 필수 조건을 말하지 않는다.
- Phase 1은 최대 5개 신호로 줄이되, 정답 태그명을 공개하지 않는다.
- Phase 2는 핵심만 먼저 보여주되, 필요한 시스템 정보는 접기 영역에서 확인 가능해야 한다.
- 신규 상태는 화면 압축 대상이더라도 누락하면 안 된다.

## 2. 용병 이력서 기본 구조

용병 이력서 첫 화면은 다음 정보만 우선 노출한다.

```text
이름 / 별칭
출신 한 줄
5대 스탯 요약
장비/임플란트 요약
대표 분석 3~5개
현재 상태
짧은 이력 텍스트
```

## 3. 용병 이력서 Phase별 압축

### Phase 0

보여줄 것:

- 이름
- 출신
- 짧은 이력 텍스트
- 현재 상태

제한:

- 이력 텍스트는 4~6문장 권장.
- 직접 태그명 금지.
- 숨겨진 만성/내재 문제는 암시만 한다.
- 신규 발생 상태는 직접 표시한다.

### Phase 1

보여줄 것:

- 5대 스탯 요약
- 대표 분석 3~5개
- 미확인 위험 표시

제한:

- 대표 분석은 최대 5개.
- 긍정 신호와 위험 신호를 섞어 보여준다.
- 모든 태그를 펼치지 않는다.

권장 형식:

```text
대표 분석:
- 강점 2~3개
- 위험 1~2개
- 미확인 0~1개
```

### Phase 2

보여줄 것:

- 시스템 태그 상세
- 만성 상태
- 내재 징크스
- 일부 공개 이야기 단서

제한:

- 기본 화면에는 분류별 요약만 보여준다.
- 상세 태그 목록은 접기 영역으로 둔다.
- 이야기 태그는 전부 자동 공개하지 않는다.

## 4. 미션 카드 기본 구조

미션 카드 첫 화면은 다음 정보만 우선 노출한다.

```text
의뢰명
위치
의뢰 성격
보상/위험도 요약
브리핑
현재 대입 용병과의 매칭 요약
```

## 5. 미션 카드 Phase별 압축

### Phase 0

보여줄 것:

- 의뢰명
- 위치
- 의뢰 성격
- 브리핑
- 보상 대략치

제한:

- 브리핑은 3~5문장 권장.
- 필수 조건을 직접 쓰지 않는다.
- 위험은 사건, 흔적, 환경으로 암시한다.

### Phase 1

보여줄 것:

- 출격 가능/불가 또는 경고
- 매칭 신호 3~5개
- 예상 성공률 또는 대략적 위험도

제한:

- 매칭 신호는 최대 5개.
- 정답 태그를 직접 공개하지 않는다.
- 위험 방향만 알려준다.

권장 형식:

```text
출격 판정:
- 가능 / 경고 / 불가

매칭 신호:
- 유리 1~2개
- 불리 1~2개
- 미확인 0~1개
```

### Phase 2

보여줄 것:

- 필수 조건
- 권장 조건
- 요구 스탯
- 위험 노드
- 조커 카드 후보
- 현재 용병과의 충돌/시너지

제한:

- 기본 화면에는 핵심 6~8개만 표시한다.
- 상세 계산식은 숨긴다.
- 위험 노드 전체 목록은 접기 가능하게 한다.

## 6. 매칭 결과 압축

매칭 결과는 3줄 요약을 기본으로 한다.

권장 구조:

```text
장점:
단점:
판단:
```

예:

```text
장점: 높은 Frame, 정비 경험
단점: 고가시성, 협소 공간 위험
판단: 핵심 작업은 가능하지만 리스크가 큼
```

## 7. 표시 수 제한

초기 기준:

- Phase 1 용병 대표 분석: 최대 5개.
- Phase 1 미션 매칭 신호: 최대 5개.
- Phase 2 미션 핵심 요구: 최대 8개.
- Phase 2 용병 기본 상세: 분류별 1~2줄.
- 결과 보고서 신규 상태: 전부 표시한다.
- 결과 보고서 핵심 로그: 성공/실패 원인을 판독할 수 있는 장면은 본문 판단 흐름에 남긴다.

## 8. 접기 영역 기준

접기 영역으로 보낼 정보:

- 전체 태그 목록
- 상세 계산식
- 위험 노드 전체 목록
- 장비/임플란트 세부 스펙
- 이야기 태그 상세
- 반복 전투 묘사나 세부 작업 동작

접기 영역으로 보내면 안 되는 정보:

- 이력서에서 성격, 장점, 위험을 판독하게 만드는 핵심 이력 문장
- 미션 브리핑에서 필요한 준비를 추론하게 만드는 핵심 현장 단서
- 결과 보고서에서 성공/실패 이유를 판독하게 만드는 핵심 사건 로그

첫 화면에 남길 정보:

- 현재 판단에 필요한 핵심 요약
- 판독에 필요한 핵심 줄글
- 신규 상태
- 출격 가능 여부
- 주요 장점/단점

## 9. 테스트 샘플 주의

`테스트 판독 샘플.md`나 `99_레거시_참고/미션 & 용병 예시.md` 같은 샘플 문서는 시스템 구조를 보여주기 위해 실제 게임 텍스트보다 직접적으로 작성될 수 있다.

실제 게임 문구는 `정보 노출 강도 규칙.md`를 따른다.

즉:

- 샘플 문서: 구조 확인용.
- 실제 Phase 0: 더 암시적.
- 실제 Phase 1: 더 압축된 신호.
- 실제 Phase 2: 명확하지만 접기 구조 사용.

## 10. 검토 질문

새 이력서나 미션 카드를 만들면 아래를 확인한다.

- 첫 화면에서 핵심 판단이 되는가?
- 읽을 정보가 너무 많지 않은가?
- 스탯과 태그가 서로 역할을 나눠 갖는가?
- 이야기 태그가 시스템 판단을 방해하지 않는가?
- Phase 0이 너무 정답지 같지 않은가?
- Phase 2가 너무 계산표처럼 보이지 않는가?
