# 정보 노출 강도 규칙

이 문서는 용병 이력서, 미션 카드, 결과 보고서에서 정보를 얼마나 직접적으로 보여줄지 정하는 기준 문서다.

현재 단계에서는 문구나 말투를 세부 확정하지 않는다. 먼저 **어떤 단계에서 얼마나 말할 것인가**를 정한다.

## 1. 핵심 원칙

- 태그는 단순하게 둔다.
- Phase 0에서는 정보를 직접 말하지 않고 암시한다.
- Phase 1에서는 정답이 아니라 신호를 준다.
- Phase 2에서는 시스템 요구와 충돌 이유를 명확히 공개한다.
- 플레이어가 직접 만든 신규 상태는 즉시 공개한다.
- 기존부터 있던 만성/내재 문제는 초반에는 숨기고, 텍스트로 암시한다.

## 1.1 다른 문서와의 관계

이 문서는 **정보를 얼마나 직접적으로 말할지**를 정하는 상위 규칙이다.

`이력서 및 미션 카드 압축 규칙.md`는 이 규칙을 바탕으로 **한 화면에 얼마나 보여줄지**를 정한다.

우선순위:

```text
정보 노출 강도 규칙
→ 이력서 및 미션 카드 압축 규칙
→ 실제 UI 문구/말투
```

따라서 화면 압축을 위해 정보를 줄이더라도, Phase별 공개 원칙을 깨면 안 된다.

예:

- Phase 0에서 필수 조건을 직접 말하면 안 된다.
- Phase 1에서 숨김 태그의 정확한 이름을 공개하면 안 된다.
- Phase 2에서 시스템 요구를 지나치게 숨기면 안 된다.
- 신규 발생 상태는 어떤 Phase에서도 숨기면 안 된다.

## 2. Phase 0: 암시

초기 단계.

플레이어는 용병 이력서와 미션 브리핑을 읽고 직접 추론한다.

### 보여주는 것

- 상황 묘사
- 사건의 흔적
- 주변 인물의 반응
- 용병의 이력과 습관
- 장비나 신체의 외형적 단서

### 숨기는 것

- 직접 태그명
- 필수 조건
- 권장 조건
- 정확한 위험 노드
- 숨겨진 만성 상태
- 숨겨진 이야기 태그

### 문장 강도

직접 말하지 않는다.

나쁜 방향:

```text
방독 장비가 필요하다.
협소 공간 징크스가 있으면 위험하다.
고가시성 장비는 적발된다.
```

좋은 방향:

```text
먼저 들어간 작업자들이 남긴 산소통 안쪽은 누렇게 변색되어 있었다.
밸브실로 가는 길은 오래된 배수관 하나뿐이다.
입구의 임시 검문대는 장비 가방을 하나씩 열어보고 있었다.
```

목표:

- 플레이어가 "아, 호흡/협소/검문 쪽 문제겠구나" 하고 추론하게 만든다.

## 3. Phase 1: 신호

분석 기관 Lv.1 단계.

정확한 답을 공개하지 않고, 매칭 결과의 방향만 알려준다.

### 보여주는 것

- 강점 신호
- 위험 신호
- 조건부 유리/불리 신호
- 미확인 위험 존재
- 출격 가능/불가 또는 경고

### 숨기는 것

- 정확한 숨김 태그명
- 모든 필수/권장 조건
- 상세 계산식
- 이야기 태그의 정체

### 권장 표시

```text
🟢 유리
🔴 불리
△ 조건부 유리
▽ 조건부 불리
❓ 미확인
⚠️ 경고
🚫 출격 불가
```

### 문장 강도

정확한 원인을 말하기보다 범주를 말한다.

예:

```text
호흡 관련 위험 감지
장비 노출 리스크 있음
미확인 내재 위험과 지형 조건 충돌 가능성
```

목표:

- 플레이어가 완전한 정답 없이도 위험 방향을 감지한다.
- 용병 분석 기관과 미션 분석 기관의 업그레이드 욕구가 생긴다.

## 4. Phase 2: 공개

분석 기관 Lv.2 단계.

이 단계에서는 시스템 정보가 명확히 공개된다.

### 보여주는 것

- 필수 조건
- 권장 조건
- 주요 위험 노드
- 요구 스탯
- 태그 충돌 이유
- 태그 시너지 이유
- 숨겨진 만성 상태
- 내재 징크스

### 여전히 숨길 수 있는 것

- 일부 이야기 태그
- 장기 스토리 핵심 비밀
- 아직 발생하지 않은 이벤트 결과
- 내부 진행 플래그

### 문장 강도

시스템 정보는 명확히 말한다.

예:

```text
필수: 호흡 보조 장비
권장: 낮은 장비 가시성
위험 노드: 협소 지형
충돌: 협소 공간 관련 내재 징크스
```

목표:

- 후반 플레이어가 최적화 매니지먼트를 할 수 있게 한다.
- 단, 이야기 비밀까지 전부 자동 공개하지는 않는다.

## 5. 결과 보고서: 발생 사실 공개

결과 보고서는 추론보다 정산에 가깝다.

플레이어가 직접 보낸 미션에서 발생한 일은 명확히 보여준다.

### 반드시 공개

- 신규 부상
- 신규 피로/스트레스
- 장비 파손
- 임플란트 손상
- 새로 생긴 수배/노출 상태
- 획득 보상
- 실패 원인

### 이유

플레이어가 직접 만든 결과를 숨기면 불합리하다.

예:

```text
신규 상태: 총상
장비 상태: 외장 소총 파손
노출 상태: 알-마디낫 검문 기록에 얼굴 일부 등록
```

## 6. 만성/내재 상태 노출

고용 시점부터 있던 문제는 초반에 숨길 수 있다.

### Phase 0

- 텍스트로만 암시.
- 직접 태그명 금지.

### Phase 1

- 미확인 위험으로 표시.
- 정확한 정체는 숨김.

### Phase 2

- 명확히 공개.

### 예시 흐름

```text
Phase 0:
좁은 곳에 들어가면 호흡이 거칠어진다는 기록이 있다.

Phase 1:
❓ 미확인 내재 위험

Phase 2:
내재 징크스: 협소 공간 공포
```

## 7. 이야기 태그 노출

이야기 태그는 시스템 분석기만으로 전부 공개하지 않는다.

### 공개 방식

- 이력서 텍스트에 단서로 암시.
- 특정 대화에서 드러남.
- 특정 미션에서 반응.
- 결과 보고서에서 후속 이벤트로 연결.

### 숨겨야 하는 것

- 반전성 과거
- 장기 분기 핵심
- 특정 NPC와의 비밀 관계
- 아직 발생하지 않은 추적/복수 이벤트

목표:

- 이야기 태그는 숫자 최적화가 아니라 캐릭터 서사와 이벤트의 재료로 남긴다.

## 8. 미션 브리핑 노출 강도

미션 브리핑은 Phase 0에서 너무 직접적이면 안 된다.

### Phase 0 브리핑

- 직접 요구 조건 대신 흔적과 상황을 보여준다.
- 플레이어가 필요한 태그군을 추론하게 한다.

### Phase 1 브리핑

- 브리핑은 유지.
- 매칭 신호가 추가된다.

### Phase 2 브리핑

- 브리핑은 유지.
- 시스템 요구 조건이 별도 영역에 공개된다.

## 9. 이력서 노출 강도

용병 이력서도 Phase 0에서 너무 정답지를 주면 안 된다.

### Phase 0 이력서

- 이력과 습관 중심.
- 장점/단점을 텍스트에 섞어 암시.

### Phase 1 이력서

- 스탯 시각화.
- 대표 분석 3~5개.
- 미확인 위험 표시 가능.

### Phase 2 이력서

- 시스템 태그 상세 공개.
- 만성 상태 공개.
- 일부 이야기 단서는 여전히 숨김 가능.

## 10. 실제 문구 작성은 후순위

이 문서는 문체 확정 문서가 아니다.

아직 정하지 않는 것:

- 미션 브리핑의 최종 말투
- 용병 이력서 문체
- 결과 보고서 문장 스타일
- UI 세부 문구

먼저 정하는 것:

- 어떤 단계에서 얼마나 직접적으로 말할지
- 무엇을 숨길지
- 무엇을 즉시 공개할지
- 무엇을 추론하게 둘지
