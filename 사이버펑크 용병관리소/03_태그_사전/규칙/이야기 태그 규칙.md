# 이야기 태그 규칙

이 문서는 이야기 태그를 만들기 위한 판정 기준이다.

실제 태그 목록을 작성하지 않는다. 어떤 태그 후보가 이야기 태그인지, 시스템 태그와 어떻게 분리할지, 언제 공개할지 정한다.

## 1. 이야기 태그 정의

이야기 태그는 캐릭터 서사, 관계, 비밀, 장기 이벤트를 관리하는 태그다.

주요 사용처:

- 용병의 과거.
- NPC와의 관계.
- 팩션과의 숨은 연결.
- 배신, 복수, 구원, 회수, 폭로 이벤트.
- 장기 스토리 분기.
- 결과 보고서 이후 후속 사건.

이야기 태그는 숫자 최적화용 정보가 아니다.

## 2. 시스템 태그와의 차이

```text
시스템 태그 = 지금 미션 판정에 직접 읽힘
이야기 태그 = 캐릭터/관계/사건 흐름을 여는 단서
```

시스템 태그는 플레이어의 즉시 파견 판단에 중요하다.

이야기 태그는 플레이어가 인물과 세계를 읽고, 장기 리스크와 기회를 예상하게 만든다.

## 3. 이야기 태그 후보 조건

아래 중 하나 이상을 만족하면 이야기 태그 후보가 된다.

- 특정 과거, 비밀, 원한, 은혜와 연결된다.
- 특정 NPC나 팩션과 특수 반응을 만든다.
- 미션 성공/실패 후 후속 이벤트로 이어진다.
- 결과 보고서의 텍스트 이유를 풍부하게 만든다.
- 장기적으로 용병의 충성, 배신, 이탈, 성장과 연결된다.
- 분석기로 전부 공개하면 캐릭터 서사가 망가진다.

## 4. 이야기 태그가 아닌 것

아래는 이야기 태그로만 두기 어렵다.

- 출격 가능/불가를 직접 결정한다.
- 특정 노드 성공률을 즉시 바꾼다.
- 장비 파손, 비용, 보험, 보상에 직접 계산된다.
- 미션 카드에서 필수/권장 조건으로 쓰인다.

이런 정보는 시스템 태그 또는 시스템 태그와의 분리 후보로 본다.

## 5. 혼합 후보 처리

어떤 태그 후보는 시스템과 이야기를 동시에 건드릴 수 있다.

이 경우 하나의 복잡한 태그로 만들지 않는다.

가능하면 둘로 나눈다.

```text
시스템 태그: 현재 판정에 필요한 짧은 조건
이야기 태그: 그 조건이 생긴 이유나 장기 사건
```

예:

```text
신체 결함 자체는 시스템 태그 후보.
그 결함이 특정 팩션의 실험 때문이라는 사실은 이야기 태그 후보.
```

## 6. 공개 원칙

이야기 태그는 시스템 분석기만으로 전부 공개하지 않는다.

공개 방식:

- 이력서 텍스트에 단서로 암시.
- 특정 미션에서 반응.
- 특정 NPC 대화에서 드러남.
- 결과 보고서에서 이상 징후로 나타남.
- 후속 이벤트에서 명확해짐.

Phase 2에서도 아래는 숨길 수 있다.

- 반전성 과거.
- 장기 분기 핵심.
- 특정 NPC와의 비밀 관계.
- 아직 발생하지 않은 추적/복수 이벤트.

## 7. 이야기 태그가 시스템에 닿는 방식

이야기 태그는 직접 수치 보정으로 남발하지 않는다.

시스템에 닿는 방식은 제한한다.

- 특정 미션에서 특수 로그 발생.
- 특정 팩션 미션에서 돌발 노드 후보 생성.
- 결과 보고서 이후 후속 미션 해금.
- 용병 충성/불만/이탈 이벤트의 조건.
- 특정 NPC와 만났을 때 선택지 변화.

## 8. 붙는 대상

이야기 태그는 주로 아래 대상에 붙는다.

- 용병.
- NPC.
- 팩션.
- 특정 장비/임플란트.
- 특정 지역.
- 미션 결과.

장비나 임플란트에도 이야기 태그가 붙을 수 있다.

단, 장비 성능이 아니라 소유 이력, 금기 실험, 회수 대상, 특정 인물과의 관계 같은 정보일 때만 이야기 태그로 본다.

## 9. 전용성 원칙

이야기 태그는 공용화에 신중해야 한다.

공용 이야기 태그 후보:

- 여러 인물에게 반복 적용 가능한 서사 기능.
- 장기 이벤트 구조로 반복 사용 가능.
- 특정 고유명 없이도 의미가 유지된다.

전용 이야기 태그 후보:

- 특정 가문, 팩션, 지역, NPC와 강하게 연결된다.
- 고유 사건이나 비밀을 품는다.
- 공용으로 만들면 서사가 평평해진다.

## 10. 작성 금지

- 이야기 태그를 전부 스탯 보너스로 바꾸지 않는다.
- 캐릭터 비밀을 Phase 2에서 자동 전부 공개하지 않는다.
- 같은 내용을 시스템 태그와 이야기 태그에 중복 기록하지 않는다.
- 모든 인물에게 과거 비밀 태그를 붙여 정보량을 폭증시키지 않는다.

## 11. 검토 질문

이야기 태그 후보마다 아래를 확인한다.

```text
1. 이 태그는 어떤 이야기를 여는가?
2. 지금 미션 판정에 직접 필요한가?
3. 시스템 태그로 분리해야 할 부분이 있는가?
4. 언제, 어떤 방식으로 플레이어에게 드러나는가?
5. 공개하면 재미가 늘어나는가, 줄어드는가?
6. 특정 지역/팩션 전용으로 내려야 하는가?
```
