# 시스템 태그 규칙

이 문서는 시스템 태그를 만들기 위한 판정 기준이다.

실제 태그 목록을 작성하지 않는다. 어떤 태그 후보가 시스템 태그인지, 어디서 읽히는지, 어느 단계에서 공개되는지만 정한다.

## 1. 시스템 태그 정의

시스템 태그는 게임 판정에 직접 쓰이는 태그다.

시스템 태그가 개입하는 곳:

- 미션 진입 조건.
- 기본 관문 돌파 확률.
- 돌발 노드 발생/완화.
- 조커 카드 발동.
- 장비/임플란트 시너지.
- 장비 가시성, 파손, 보험, 유지비.
- 보상, 평판, 수배, 신규 상태.

시스템 태그는 플레이어가 파견 판단을 하게 만드는 정보다.

## 2. 핵심 원칙

태그는 단순해야 한다.

복잡한 판정은 미션, 장비, 스테이션, 결과 보고서가 해석한다.

```text
태그 = 짧은 속성
미션 = 그 속성을 상황별로 해석
결과 보고서 = 왜 그런 결과가 났는지 설명
```

## 3. 시스템 태그 후보 조건

아래 중 하나 이상을 만족하면 시스템 태그 후보가 된다.

- 출격 가능/불가에 영향을 준다.
- 미션 성공률, 위험도, 소요 시간에 영향을 준다.
- 특정 노드에서 유리/불리/무시/특수 반응으로 읽힌다.
- 장비나 임플란트의 성능, 가시성, 파손, 수리비에 영향을 준다.
- 스테이션의 준비 효과와 상호작용한다.
- 결과 보고서에서 신규 상태나 비용으로 정산된다.

## 4. 시스템 태그가 아닌 것

아래는 시스템 태그로 바로 올리지 않는다.

- 캐릭터의 과거 비밀만 설명하는 정보.
- 숫자 판정 없이 대화나 사건 분기에만 쓰이는 정보.
- 특정 인물의 정체성 설명.
- 아직 게임 내 선택이나 결과와 연결되지 않은 세계관 설정.

이런 정보는 이야기 태그 후보로 보낸다.

## 5. 읽히는 위치

시스템 태그는 최소 하나의 읽는 위치를 가져야 한다.

```text
미션 카드
용병 이력서
장비 상세
임플란트 상세
스테이션 준비
미션 노드
결과 보고서
```

읽는 위치가 없으면 태그가 아니라 설정 메모다.

## 6. 판정 축

시스템 태그는 다음 축 중 하나 이상으로 판정한다.

### 진입 조건

미션에 들어갈 수 있는지 판단한다.

예:

- 특정 환경 대응 장비 필요.
- 특정 가시성 이하 필요.
- 특정 임플란트 파츠 정상 작동 필요.

### 관문 보정

기본 관문에서 성공률, 소요 시간, 위험도를 바꾼다.

### 돌발 노드

조건 부족이나 과잉, 숨은 위험, 부정 상태가 돌발 노드를 만든다.

### 조커 카드

실패했을 때 서브 스탯, 내장 무기, 임플란트, 스테이션 효과가 실패를 무마한다.

### 정산

결과 보고서에서 부상, 파손, 수배, 보험, 재화로 정산된다.

## 7. 공개 단계

시스템 태그 공개는 `정보 노출 강도 규칙.md`를 따른다.

- Phase 0: 직접 태그명 금지. 텍스트로 암시.
- Phase 1: 태그명 대신 신호와 범주.
- Phase 2: 시스템 요구, 충돌, 시너지 이유 공개.
- 신규 상태: 즉시 공개.

시스템 태그라고 해서 초반부터 전부 공개하지 않는다.

## 8. 공용/전용 판정

공용 시스템 태그 후보:

- 여러 대상에서 같은 시스템 판단을 만든다.
- 미션 해석 규칙에서 반복 사용된다.
- 특정 지역/팩션 고유명 없이도 의미가 유지된다.

전용 시스템 태그 후보:

- 특정 지역, 팩션, 장비, 임플란트, 용병군에서만 의미가 선명하다.
- 공용으로 올리면 의미가 흐려진다.
- 세계관 고유 구조와 강하게 묶인다.

## 9. 작성 금지

- 태그명 안에 수치 효과를 넣지 않는다.
- 한 태그에 성공률, 비용, 이야기 분기를 전부 넣지 않는다.
- 긍정/부정으로만 고정하지 않는다.
- 모든 미션에서 같은 의미로 작동한다고 가정하지 않는다.

## 10. 검토 질문

시스템 태그 후보마다 아래를 확인한다.

```text
1. 어디에 붙는가?
2. 어느 시스템이 읽는가?
3. 플레이어가 어떤 판단을 하게 되는가?
4. 어느 Phase에서 어떻게 보이는가?
5. 미션마다 해석이 달라질 여지가 있는가?
6. 이야기 태그와 섞여 있지는 않은가?
```
