1. 미션 화면
	- 의뢰 목록이 카드 형태로 표기
	- 각 카드는 정사각형에 테두리 색으로 어느 구역인지 표시(하층은 시안, 중층은 보라, 상층은 마젠타)
	- 화면 구성
		- 좌측 사이드바에 메뉴바 배치(다른 메뉴나 설정 창 등으로 이동)
		- 사이드바를 제외한 화면에 미션 카드 배치(카드 크기가 전체적으로 너무 꽉 막히지 않는 선에서 화면에 들어차게 가변하도록 설정)
		- 카드 배치는 가로 3개 배치가 기본이며, 3개 초과는 세로 스크롤로 확인
		- 기본 배치는 좌측 상단을 기준으로 지역별 나열(하층, 중층, 상층 순으로 상단에 배치)
	- 미션은 매 3시간마다 갱신
	- 갱신 때마다 총 미션 개수 및 종류, 각 구역별 미션 개수 등이 변화(개수의 경우 테스트 버전에서는 최소 3개, 최대 12개로 제한)
	- MVP 테스트 확인 항목
		- 테스트 시작 시 미션 3개가 게시판에 정상 표시되는지 확인
		- 각 미션 카드에서 구역, 미션 타입, 난이도, 보상 등 최소 식별 정보가 확인되는지 점검
		- 지역별 테두리 색이 하층/중층/상층 구분에 충분히 사용 가능한지 확인
		- 3개 초과 미션이 있을 경우 스크롤로 추가 미션을 확인할 수 있는지 점검

2. 미션 확인
	- 미션 화면에서 미션을 클릭할 경우 나오는 화면
	- 미션의 상세 화면이며, 이때 미션에 대한 판단을 플레이어가 실행
	- 미션 분석 기관의 레벨에 따라 공개되는 정보 변동
	- 화면 구성
		- 미션 화면의 좌측 사이드바 유지
		- 의뢰 서류철 같은 느낌의 창에 미션 정보가 적혀 있는 화면
			- 좌측 상단에 의뢰 관련 시각 정보(미션 목표의 사진이라던가 관련 기사라던가 등)나 물품 첨부(현장 흔적이라던가 증거 샘플이라던가 등) 느낌(사펑 느낌 나는 칩이나 실제 물품 등으로 표기, 클릭하면 시각 정보나 물품 상세 정보 팝업)
			- 중앙 및 우측 상단에 상부표를 적고 미션 타입, 대략적인 미션 목표, 상세 위치, 난이도, 보상 표기(이 부분과 브리핑 부분은 사이버 펑크 2077의 픽서 의뢰 문자로 오는 것을 반영할 것)
			- 상단 부분을 제외한 나머지 공간에 브리핑 표기
			- 분석 기관 레벨에 따라 추가 페이지로 정보 추가, 드래그 혹은 첫 페이지 우측 테두리 부근 클릭으로 페이지 넘김
		- 우측 하단에 돌아가기 버튼이 생기며, 누르면 미션 화면으로 복귀
		- 내용이 길면 페이지가 넘어감. 이때는 추가 정보 페이지와는 별개로 내용이 길어진 부근의 양식을 이어서 사용
	- 미션의 지역에 따른 색은 서류철 색으로 반영, 미션 정보가 적힌 곳은 배경색 흰색으로 유지
	- 미션 확인 창의 서류 아랫부분에 서명하면 미션이 수주되고, 수주한 미션은 수주 미션 목록으로 넘어가며 미션 화면에서 사라짐
	- MVP 1턴 수동 테스트에서는 수주 후 취소 기능을 제외함
	- MVP 테스트 확인 항목
		- 미션 카드 클릭 후 상세 화면으로 정상 전환되는지 확인
		- 미션 분석 기관 레벨에 따라 추가 정보 페이지가 열리거나 숨겨지는지 확인
		- 돌아가기 버튼을 눌렀을 때 미션 화면으로 복귀하는지 확인
		- 서명 후 해당 미션이 게시판에서 사라지고 수주 미션 목록으로 이동하는지 확인
		- 수주 후 취소 기능은 MVP 테스트 필수 확인 항목에서 제외
		- 수주 전후에 미션 정보가 섞이거나 손실되지 않는지 확인

3. 미션 매칭
	- 수주한 미션 목록이 먼저 뜸. 기본 나열은 수주한 순서대로 최근 수주가 위로 오게 배치
	- 여기서의 미션 목록은 미션 화면하고 기본 화면은 같으나, 미션 카드를 살짝 어둡게 만들고 그 위에 아주 간략하게 미션 타입, 난이도 표시
	- 여기는 이미 수주한 의뢰 목록이므로 상세 필터 존재. 우측 상단에 필터로 난이도, 구역, 미션 타입 등으로 필터링 및 순서 재배치 가능
	- 미션 카드 클릭하면 앞선 미션 확인 버전에서 브리핑을 제거한 버전의 요약 화면이 나오고, 그 화면의 우측 하단의 버튼을 누르면 브리핑 및 분석 기관 레벨에 맞춘 추가 정보창이 나옴
	- 요약 화면을 누르면 용병 매칭 화면으로 전환. 화면 크기를 대략 6:4로 위가 6이 되게 나눈 후, 아래에는 용병 목록 및 간략 요약 화면이 나오고 위에는 좌측에 용병이 들어가는 슬롯 느낌의 공간 배치. 그 슬롯이 채워지면 위 화면의 우측에는 매칭 분석 정도가 분석 기관 레벨에 맞춰 표시
	- 용병을 매칭하고 아래 화면에 밀어서 잠금해제 같은 슬라이드 버튼을 슬라이드하면 미션 스타트
	- MVP 테스트 확인 항목
		- 수주한 미션 목록에서 선택한 미션의 요약 정보가 정상 표시되는지 확인
		- 용병 목록에서 테스트 용병 4명이 확인되는지 점검
		- 용병을 미션 슬롯에 배치했을 때 매칭 분석 영역이 갱신되는지 확인
		- 매칭 분석 정도가 미션 분석 기관 레벨과 용병 분석 기관 레벨의 조합을 기준으로 표시되는지 확인
		- 유리, 불리, 조건부, 미확인, 출격 가능, 경고, 불가 같은 판단 신호가 테스트 케이스에 맞게 확인되는지 점검
		- 출격 전 최종 조합이 어떤 미션과 어떤 용병인지 플레이어가 다시 확인할 수 있는지 점검

4. 미션 진행 화면
	- 이 화면에서는 유저가 선택한 모드에 따라 다른 화면이 보임
	- MVP 1턴 수동 테스트에서는 우선 어사인 방식으로 출격 후 결과 보고서까지 이어지는 흐름을 확인
	- 캐치업은 후속 테스트 또는 MVP 이후 확장 모드로 분리하여 검토
	- 모드는 두 가지로 구상
		1. 드론으로 따라가면서 노드 진행을 관찰하고 변수 통제를 일부 지원할 수 있으나, 난이도가 조금 올라가고 미션 실패 시 리스크가 더욱 커지는 캐치업
		2. 용병에게 모든 것을 맡기고 노드 발생, 완료 시에만 해당 정보를 습득 가능하고 현장 변수 통제는 사전 준비(용병 관리 스테이션 기능 사용 등)를 빼고 못하지만 리스크가 적어지는 어사인
	- 캐치업
		- 총 노드 중 3할의 노드에 간접적으로 관여 가능
		- 관여 가능 노드 발생 시 선택지가 발생하며 유저가 고른 대로 노드 진행
		- 화면 구성
			- 기본적으로 좌측 사이드바는 유지, 남은 화면 5:5로 위아래 분할
			- 위에는 그래픽으로 용병을 TPS 느낌으로 따라가는 드론의 화면
				- 그래픽은 구역과 미션 타입, 진행 노드 등의 변수에 따라 변경됨(테스트에서는 미구현이어도 상관은 없으나, 구현한다면 미션 타입과 구역만 반영할 것)
				- 기본은 이동하는 화면, 노드 발생 시 정지, 노드 처리, 이동 반복
			- 하부는 노드 로그로 노드 시작 시 어떤 노드인지, 어떤 식으로 해결했는지, 실패했는지 등 보고서로 받는 내용을 미리 받는 느낌의 내용을 좀 더 생동감 있게 적은 내용이 표시됨. 약간 채팅 로그처럼 옛날 글이 아래로 내려가는 느낌
		- 캐치업에서는 기본적으로 실패나 도중 철수 등의 리스크 발생 시의 비용이 1.5배이며, 어사인과 달리 부정적 상황 발생 확률이 15% 정도 높음. 대신 선택지 조정으로 보수를 더 많이 받는다거나 특별한 보상을 찾을 확률이 어사인보다 높음
	- 어사인
		- 용병 출격 후 총 노드 중 돌발 노드 발생 시에만 알림 발생
		- 그 외의 노드 진행은 추후 보고서로 수령
		- MVP 1턴 수동 테스트에서는 어사인 진행 중 유저 개입 없이 관찰만 진행
		- 화면 구성
			- 기본적으로 좌측 사이드바는 유지, 남은 화면 3:7로 위아래 분할
			- 상단 부분에는 모니터를 보고 있는 유저 캐릭터
				- 일반적으로 푸른 빛의 홀로그램 모니터를 보고 있는 장면(좌측에 플레이어 픽서 캐릭터, 우측에 모니터 여러 대 = 마치 주식 분석가 같은 느낌의 모니터 배치)
				- 돌발 발생 시 모니터를 너무 진하지 않은 붉은색으로 변경, 캐릭터도 몸을 모니터 쪽으로 가까이하고 긴장한 태도
			- 하부는 용병의 남은 HP와 장비 상태(이 정도의 모니터링은 지금까지의 로어를 무너뜨리지 않는다고 판단), 진행도 표시. 노드 진행 시마다 갱신
		- 어사인에서는 기본적으로 리스크 발생 시의 비용은 그대로 내고 부정적 상황 발생 확률의 별도 변동은 없음. 대신 추가 보상이나 특별 보상 찾을 확률이 낮음
	- MVP 테스트 확인 항목
		- 1턴 수동 테스트에서는 어사인 방식으로 출격 후 결과 보고서로 넘어가는 흐름을 우선 확인
		- 출격 후 진행 상태, 돌발 노드 알림, 진행도 표시가 최소한의 확인 가능한 형태로 연결되는지 점검
		- 어사인 진행 중 선택지 개입이나 수동 조작은 발생하지 않는지 확인
		- 캐치업 관련 선택지 개입, 확률 변화, 리스크 가중치는 이번 MVP 수동 테스트의 필수 확인 항목에서 제외

5. 미션 결과 화면
	- 캐치업과 어사인 둘 다 보고서를 받는 화면
		- 다만 문서의 양식이 캐치업은 픽서가 직접 보고 기록한 방식, 어사인은 용병이 보고하는 방식
	- 화면 구성
		- 내용의 표현과 양식은 캐치업과 어사인이 다르지만 기본적인 화면 디자인은 동일
		- 수행한 미션 카드의 테두리 색과 동일한 색의 바탕에 노드 발생과 진행과 완료가 정리된 내용, 그 아래에 좌측에 성공 실패 여부, 우측에 보상과 발생 리스크, 수리비 등의 정산 내용
		- MVP 1턴 수동 테스트에서는 하단 버튼을 바로 정산 하나로 단순화
	- MVP 테스트 확인 항목
		- 결과 보고서에서 성공, 부분 성공, 실패, 조기 철수, 사고 중 해당 결과가 명확히 표시되는지 확인
		- 보상, 발생 리스크, 수리비, 손상, 평판 변화 등 정산 정보가 확인되는지 점검
		- 바로 정산 후 용병 상태, 장비 상태, 평판, 보상 기록에 반영되는지 확인
		- 결과 보고서 내용이 이전 단계의 미션, 용병, 매칭 판단과 이어지는지 점검
