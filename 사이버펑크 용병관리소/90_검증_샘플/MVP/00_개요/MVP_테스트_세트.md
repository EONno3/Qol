# MVP 테스트 세트

이 문서는 MVP 단계에서 실제 게임 루프가 작동하는지 검증하기 위한 최소 테스트 세트다.

목표는 모든 설정을 확정하는 것이 아니라, 아래 한 사이클이 수동 데이터만으로 끊기지 않고 이어지는지 확인하는 것이다.

```text
미션 게시판 확인
-> 용병/미션 Phase 정보 확인
-> 미션 슬롯 매칭
-> 출격 가능 여부 판단
-> 노드 판정
-> 성공/실패/철수 처리
-> 결과 보고서와 상태 변화 정산
```

---

## 1. MVP 테스트 원칙

- 새 규칙을 늘리기보다 기존 규칙이 실제로 맞물리는지 본다.
- 미션 카드, 용병 이력서, 결과 보고서는 우선 수동 데이터로 작성한다.
- 용병 분석 기관과 미션 분석 기관은 `Lv.0~Lv.2`까지만 사용한다.
- 미션 슬롯 매칭은 항상 `min(용병 분석 레벨, 미션 분석 레벨)`을 따른다.
- 테스트 태그와 수치는 더미가 될 수 있지만, 판단 구조는 최신 규칙을 따라야 한다.
- 로컬 AI 브리핑 생성, 대량 태그 사전화, 세부 경제 밸런스는 MVP 테스트 범위에서 제외한다.

## 2. 확정 테스트 미션

### 2.1 하층 기준 미션

- 대상 문서: `04_미션/04_하층_변전소_캐패시터_탈취.md`
- 검증 목적: 하층 환경 위험, 필수 보호 장비, 장비 미충족 시 상태 손상 검증.
- 핵심 판정 축: `Frame`, `Wire`
- 핵심 장비: 절연 안전화 계열 기어
- 핵심 위험: 감전, 의체 내구도 마손, 정산금 감산

### 2.2 중층 기준 미션

- 대상 문서: `04_미션/02_중층_클럽_엘네온_강탈.md`
- 검증 목적: 중층 보안/갱단/가시성 위험과 은닉 세팅 검증.
- 핵심 판정 축: `Cypher`, `Cool`
- 핵심 장비: 저가시성 무장, 은닉형 슬롯
- 핵심 위험: 검문 적발, 갱단 교전, 장비 파손

### 2.3 상층 기준 미션

- 대상 문서: `04_미션/03_상층_펜트하우스_스파이.md`
- 검증 목적: 상층 고위험 침투, 필수 기어 2종, 실패 시 생존 위험 검증.
- 핵심 판정 축: `Cool`, `Cypher`
- 핵심 장비: 상층 고고도 기압 보정기, 인공 DNA 유전자 스킨 패치
- 핵심 위험: 바이오메트릭 적발, 상층 평판 붕괴, 사망 확률 연산

### 2.4 보조 샘플

- 대상 문서: `04_미션/01_하층_유앤존_폐기물_인양.md`
- 용도: 하층 환경 위험의 추가 비교 샘플.
- MVP 핵심 세트에는 넣지 않는다.
- 이유: 하층 미션을 2개 모두 넣으면 MVP 검증 범위가 불필요하게 커진다.

## 3. 테스트 용병 프로필

정식 용병 사전이 아니라 MVP 루프 검증용 수동 프로필이다.

세부 이력서는 `90_검증_샘플/MVP/01_입력_데이터/MVP_테스트_용병_이력서.md`를 기준으로 한다.

미션 3개와 용병 4명의 실제 조합 판정은 `90_검증_샘플/MVP/02_판정_케이스/MVP_매칭_케이스표.md`를 기준으로 한다.

파견 후 성공/부분 성공/실패 결과 샘플은 `90_검증_샘플/MVP/02_판정_케이스/MVP_결과_보고서_샘플.md`를 기준으로 한다.

### 3.1 하층 적합 프로필

- 기준 용병: 백건우 / `차단기`
- 역할: 전력망/산업 현장 처리 담당.
- 강점: `Frame`, `Wire`
- 대표 태그 후보: `#고압망_기술`, `#슬럼_출신`
- 장비 조건: 절연 안전화 계열 기어 보유.
- 검증 포인트: 하층 변전소 미션에서 유리 신호와 장비 충족이 정상 표시되는지 확인한다.

### 3.2 중층 적합 프로필

- 기준 용병: 한서윤 / `몰트`
- 역할: 검문 회피/은닉 침투 담당.
- 강점: `Cypher`, `Cool`
- 대표 태그 후보: `#시민_위장`, `#스캐너_기만`
- 장비 조건: 저가시성 무장, 은닉형 슬롯 보유.
- 검증 포인트: 중층 클럽 미션에서 가시성 조건과 은닉 세팅이 정상 판정되는지 확인한다.

### 3.3 상층 적합 프로필

- 기준 용병: 라일라 라쉬드 / `벨벳 나이프`
- 역할: 사교 위장/고위험 침투 담당.
- 강점: `Cool`, `Cypher`
- 대표 태그 후보: `#유전자_기만`, `#단정한_인상`
- 장비 조건: 상층 고고도 기압 보정기, 인공 DNA 유전자 스킨 패치 보유.
- 검증 포인트: 상층 펜트하우스 미션에서 필수 기어 2종과 고난도 스탯 요구가 정상 판정되는지 확인한다.

### 3.4 실패 유도 프로필

- 기준 용병: 도마 준 / `크롬쇼`
- 역할: 실패/경고/철수/결과 보고서 검증용.
- 약점: 미션 요구 축과 맞지 않는 스탯, 과도한 장비 가시성, 필수 기어 미보유.
- 대표 태그 후보: `#과시형_크롬`, `#누전_과민`, `#하층_슬랭_고착`
- 검증 포인트: 출격 경고, 조건 미충족, 위험 노드, 상태 손상, 정산 손실이 정상 출력되는지 확인한다.

## 4. 필수 검증 항목

### 4.1 Phase 정보 노출

- 미션 분석 Lv.0에서는 Phase 0 미션 카드만 보여야 한다.
- 미션 분석 Lv.1에서는 추천 스탯 체급, 장비 단서, 위험 방향 신호가 추가되어야 한다.
- 미션 분석 Lv.2에서는 요구 수치, 필수 기어, 보너스 조건, 위협 프로필 요약이 추가되어야 한다.
- 용병 이력서는 용병 분석 기관 레벨만 따른다.

### 4.2 매칭 해석

- 용병 분석 Lv.1 / 미션 분석 Lv.0이면 매칭 정보는 열리지 않는다.
- 용병 분석 Lv.1 / 미션 분석 Lv.1이면 방향성 신호가 열린다.
- 용병 분석 Lv.2 / 미션 분석 Lv.2이면 충돌/시너지 이유와 예상 손익이 열린다.
- 매칭 결과는 미션 카드나 용병 이력서 자체 정보와 섞이지 않는다.

### 4.3 출격 판정

- 필수 기어가 없으면 출격 불가 또는 치명적 경고가 떠야 한다.
- 권장 스탯 미달은 즉시 금지가 아니라 성공률 저하와 위험 증가로 처리한다.
- 장비 `Visibility`는 중층 미션에서 명확한 판단 요소로 작동해야 한다.

### 4.4 결과 정산

- 성공 시 보상, 평판 변화, 획득물이 기록되어야 한다.
- 실패 시 보상 없음, 평판 손실, 상태 손상 가능성이 기록되어야 한다.
- 조기 철수 시 패널티 비용이 정산되어야 한다.
- 실패 후 생존 판정은 구역별 위협 성격을 반영해야 한다.

## 5. MVP 제외 항목

아래 항목은 이번 테스트 세트에서 확정하지 않는다.

- 모든 용병 태그의 최종 명칭.
- 모든 장비와 임플란트의 대량 사전.
- 전체 경제 밸런스와 장기 수익률.
- 로컬 AI를 통한 브리핑 자동 생성.
- 액티브 분석 슬롯의 상세 비용과 대기 시간.
- 모든 노드의 세부 확률식.
- UI 세부 연출.

---

## 6. 완료 기준

MVP 테스트 세트는 아래 조건을 만족하면 통과로 본다.

- 하층/중층/상층 미션을 각각 1개씩 선택해 출격 판단을 할 수 있다.
- 적합 용병과 부적합 용병의 차이가 Phase와 매칭 화면에서 드러난다.
- 성공, 실패, 조기 철수 중 최소 2종 이상의 결과 보고서가 작성 가능하다.
- 상태 변화와 장비 손상이 결과 보고서에서 이력서/정비 판단으로 이어진다.
- 플레이어가 “왜 이 용병을 보내야 하는지”와 “왜 보내면 안 되는지”를 읽고 판단할 수 있다.
