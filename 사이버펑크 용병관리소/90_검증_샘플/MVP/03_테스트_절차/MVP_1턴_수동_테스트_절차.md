# MVP 1턴 수동 테스트 절차

이 문서는 MVP 문서군을 바탕으로 한 턴을 수동으로 굴려 보는 절차서다.

목적은 전체 게임 UI를 검증하는 것이 아니라, 아래 미션 루프가 실제로 끊기지 않는지 확인하는 것이다.

```text
미션 게시판
-> 미션 확인/수주
-> 수주 미션 목록
-> 용병 매칭
-> 어사인 출격
-> 결과 보고서
-> 바로 정산
-> 상태/장비/평판 반영
```

기준 UI 흐름은 `UI/미션 화면 흐름.md`를 따른다.

---

## 1. 테스트 시작 조건

### 1.1 기본 상태

- 플레이어는 이미 미션 게시판 화면에 진입해 있다.
- 메인 화면, 스테이션 홈, 상점, 정비소, 고용소는 이번 테스트 범위에서 제외한다.
- 미션 게시판에는 MVP 대표 미션 3개가 노출되어 있다.
- 테스트 용병 4명이 용병 목록에 준비되어 있다.
- 분석 기관 레벨은 테스트 케이스에 따라 `Lv.0~Lv.2`로 바꿔 볼 수 있다.

### 1.2 사용하는 문서

- 테스트 세트: `90_검증_샘플/MVP/00_개요/MVP_테스트_세트.md`

## 2. 테스트 대상 데이터

### 2.1 미션 3개

```text
하층: 더 퓨즈 제3변전소 고압 캐패시터 절도
중층: 클럽 엘 네온 백도어 강탈
상층: 알-마디낫 후계자 전뇌 말소
```

### 2.2 용병 4명

```text
차단기: 하층 산업/전력망형
몰트: 중층 은닉/검문 회피형
벨벳 나이프: 상층 사교/위장 침투형
크롬쇼: 실패/경고/정산 손실 유도형
```

### 2.3 결과 타입 5종

```text
success: 성공
partial_success: 부분 성공
failure: 실패
early_withdrawal: 조기 철수
incident: 사고
```

## 3. 1턴 테스트 절차

### 3.1 미션 게시판 확인

절차:

1. 미션 게시판 화면을 연다.
2. 미션 카드 3개가 카드 형태로 표시되는지 확인한다.
3. 각 카드의 테두리 색으로 구역이 구분되는지 확인한다.
4. 카드에서 최소 식별 정보가 보이는지 확인한다.

확인 항목:

- 하층은 시안, 중층은 보라, 상층은 마젠타 계열로 구분된다.
- 미션 타입, 난이도, 보상, 구역이 표시된다.
- 카드 3개가 기본 가로 배치로 읽힌다.
- 3개 초과 미션이 있을 경우 세로 스크롤로 확인 가능하다.

통과 기준:

- 플레이어가 카드만 보고 어느 구역의 어떤 위험도 미션인지 대략 구분할 수 있다.

---

### 3.2 미션 확인 및 수주

절차:

1. 미션 카드 하나를 클릭한다.
2. 의뢰 서류철 형태의 미션 상세 화면으로 전환되는지 확인한다.
3. 미션 분석 기관 레벨에 따라 Phase 정보가 열리거나 숨겨지는지 확인한다.
4. 돌아가기 버튼으로 게시판 복귀가 가능한지 확인한다.
5. 서명하여 미션을 수주한다.
6. 수주한 미션이 게시판에서 사라지고 수주 미션 목록으로 이동하는지 확인한다.

확인 항목:

- Phase 0에서는 기본 계약 정보와 브리핑 중심으로 보인다.
- Phase 1/2 정보는 미션 분석 기관 레벨에 따라 추가 페이지로 열린다.
- 수주 전후 정보가 섞이거나 손실되지 않는다.
- 수주 후 취소 기능은 MVP 테스트에서 제외한다.

통과 기준:

- 플레이어가 게시판에서 미션을 고르고, 상세를 확인한 뒤, 수주 상태로 넘길 수 있다.

---

### 3.3 수주 미션 목록 확인

절차:

1. 수주 미션 목록을 연다.
2. 최근 수주 미션이 위쪽에 표시되는지 확인한다.
3. 수주 미션 카드가 게시판 카드보다 어둡게 보이는지 확인한다.
4. 미션 타입, 난이도 같은 요약 정보가 유지되는지 확인한다.
5. 필터가 있다면 난이도, 구역, 미션 타입으로 정렬 가능 여부를 확인한다.

확인 항목:

- 수주한 미션과 게시판 미션이 구분된다.
- 상세 브리핑을 다시 열 수 있다.
- 매칭 화면으로 넘어갈 미션을 명확히 고를 수 있다.

통과 기준:

- 수주한 의뢰 목록에서 테스트할 미션을 안정적으로 다시 선택할 수 있다.

---

### 3.4 용병 매칭

절차:

1. 수주 미션을 선택해 요약 화면을 연다.
2. 요약 화면에서 용병 매칭 화면으로 전환한다.
3. 아래 영역에서 테스트 용병 4명이 보이는지 확인한다.
4. 용병을 상단 슬롯에 배치한다.
5. 슬롯이 채워지면 우측 매칭 분석 영역이 갱신되는지 확인한다.
6. 분석 레벨 조합에 따른 매칭 표시 레벨을 확인한다.

확인 항목:

- 매칭 표시 레벨은 `min(용병 분석 레벨, 미션 분석 레벨)`을 따른다.
- 매칭 Lv.0에서는 유리/불리/조건부 신호가 직접 표시되지 않는다.
- 매칭 Lv.1에서는 유리, 불리, 조건부, 미확인 신호가 표시된다.
- 매칭 Lv.2에서는 충돌/시너지 이유와 예상 손익 방향이 표시된다.
- 출격 판정은 `가능`, `경고`, `불가` 중 하나로 표시된다.

테스트 권장 조합:

```text
차단기 x 하층 변전소: 가능
몰트 x 중층 클럽: 가능
벨벳 나이프 x 상층 펜트하우스: 가능
크롬쇼 x 중층 클럽: 불가 또는 치명적 경고
```

통과 기준:

- 플레이어가 왜 이 용병을 보내야 하는지, 왜 보내면 안 되는지 매칭 화면에서 읽을 수 있다.

---

### 3.5 어사인 출격

절차:

1. 최종 미션과 용병 조합을 확인한다.
2. 슬라이드 버튼으로 출격을 시작한다.
3. 진행 화면이 어사인 모드로 전환되는지 확인한다.
4. 진행도, 용병 HP, 장비 상태가 최소한의 형태로 표시되는지 확인한다.
5. 돌발 노드가 발생하면 알림이 뜨는지 확인한다.
6. 진행이 끝나면 결과 보고서 화면으로 넘어간다.

확인 항목:

- MVP 1턴 수동 테스트에서는 어사인 방식만 사용한다.
- 어사인 진행 중 유저의 선택지 개입이나 수동 조작은 없다.
- 캐치업 모드는 후속 테스트 또는 MVP 이후 확장으로 둔다.
- 캐치업의 리스크 가중, 선택지 개입, 보상 증가 규칙은 이번 테스트에서 제외한다.

통과 기준:

- 출격 후 유저 개입 없이 진행 상태를 관찰하고, 결과 화면까지 도달할 수 있다.

---

### 3.6 결과 보고서 확인

절차:

1. 결과 보고서 화면이 열린다.
2. 결과 타입이 명확히 표시되는지 확인한다.
3. 노드 발생, 진행, 완료 내용이 보고서 형태로 표시되는지 확인한다.
4. 보상, 손실, 리스크, 수리비, 상태 변화, 평판 변화가 표시되는지 확인한다.
5. 결과 보고서 내용이 이전 미션/용병/매칭 판단과 연결되는지 확인한다.

확인 항목:

- 결과 타입은 5종 중 하나다.
- 결과 로그는 단순 분위기 문장이 아니라 성공/실패 이유를 설명한다.
- 손실과 추가 베네핏 미획득은 분리된다.
- 상태 변화와 장비 손상은 숨기지 않는다.

통과 기준:

- 플레이어가 결과 보고서를 읽고 다음에는 무엇을 준비해야 하는지 알 수 있다.

---

### 3.7 바로 정산

절차:

1. 결과 보고서 하단의 바로 정산 버튼을 누른다.
2. 보상과 손실이 반영되는지 확인한다.
3. 용병 현재 상태가 갱신되는지 확인한다.
4. 장비/임플란트 상태가 갱신되는지 확인한다.
5. 팩션 평판과 후속 단서가 반영되는지 확인한다.

확인 항목:

- MVP에서는 결과 화면 버튼을 바로 정산 하나로 단순화한다.
- 바로 정산 후 결과가 용병 이력서, 장비 상태, 평판, 후속 의뢰 후보 중 최소 하나에 반영된다.
- 정산 전후 값이 섞이거나 중복 적용되지 않는다.

통과 기준:

- 결과 보고서가 단순 로그로 끝나지 않고 다음 턴의 상태로 되돌아간다.

## 4. 테스트 케이스 3종

### 4.1 성공 케이스

```text
미션: 더 퓨즈 제3변전소 고압 캐패시터 절도
용병: 차단기
예상 결과: success
기준 결과 보고서: 차단기 x 하층 변전소
```

확인:

- 필수 장비 충족.
- 고압망 작업 경험이 성공 로그와 연결.
- 절연 장비 경미 마모만 발생.

### 4.2 부분 성공 케이스

```text
미션: 클럽 엘 네온 백도어 강탈
용병: 몰트
예상 결과: partial_success
기준 결과 보고서: 몰트 x 중층 클럽
```

확인:

- 미션 목표는 달성.
- 채무 중재금과 수면 부족 심화 발생.
- 추가 베네핏 미획득이 손실과 분리.

### 4.3 실패 케이스

```text
미션: 클럽 엘 네온 백도어 강탈
용병: 크롬쇼
예상 결과: failure
기준 결과 보고서: 크롬쇼 x 중층 클럽
```

확인:

- 높은 `Visibility`가 실패 원인으로 연결.
- 출격 전 경고/불가 신호와 결과 보고서 원인이 이어짐.
- 장비 손상, 상태 악화, 평판 손실이 정산에 반영.

## 5. 테스트 통과 기준

MVP 1턴 수동 테스트는 아래 조건을 만족하면 통과로 본다.

- 미션 3개가 게시판에 정상 표시된다.
- 미션 확인 화면에서 Phase 정보가 분석 기관 레벨에 따라 열리고 닫힌다.
- 수주한 미션이 게시판에서 사라지고 수주 미션 목록으로 이동한다.
- 용병 4명을 매칭 슬롯에 배치할 수 있다.
- 매칭 결과가 `min(용병 분석 레벨, 미션 분석 레벨)`을 따른다.
- 출격 판정이 `가능`, `경고`, `불가`로 구분된다.
- 어사인 출격 후 유저 개입 없이 결과 보고서로 이어진다.
- 결과 타입 5종 중 하나가 명확히 표시된다.
- 바로 정산 후 상태/장비/평판/보상 기록이 반영된다.
- 결과 보고서의 원인이 미션/용병/매칭 단계와 이어진다.

## 6. 이번 MVP 테스트에서 제외할 것

- 메인 화면부터 스테이션 전체 루프.
- 수주 후 취소 기능.
- 캐치업 모드 조작.
- 캐치업 리스크/보상 가중 규칙.
- 상점, 정비소, 고용소 UI.
- 전체 장비/태그/상태 사전 확정.
- 세부 노드 확률식.
- 전체 경제 밸런스.
