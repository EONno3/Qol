# 테스트 결과 보고서 2차 샘플

이 문서는 결과 보고서 구조를 다른 미션 유형에서도 검증하기 위한 2차 샘플이다.

중요:

- 태그명, 수치, 보상, 성공률은 전부 **가칭/검증용 더미 데이터**다.
- 실제 태그 사전 확정본이 아니다.
- 기준 미션은 `테스트 미션 및 용병 3차 샘플.md`의 `실버램프 야간 경비 대행`이다.
- 목적은 경비/오인식별/제한 교전 미션에서 결과 보고서가 작동하는지 확인하는 것이다.

## 1. 테스트 목표

- 전투가 적은 경비 미션도 결과 보고서로 재미있게 읽히는가?
- 오인 경보, 출입 기록, 제한 교전, 무사고 유지가 사건 로그로 드러나는가?
- `Cypher`, `Cool`, `Visibility`, `Frame` 판정이 미션/용병/보고서 사이에서 연결되는가?
- 상태 태그 변화와 장비/임플란트 변화가 섞이지 않는가?
- 긍정/조건부 상태 태그가 결과 보고서에서 작동 가능한가?

## 2. 내부 결과 설계

### 기본 정보

- 미션명: 실버램프 야간 경비 대행
- 투입 용병: 서라 / 배지
- 미션 타입: 경비 / 오인식별 위험 / 제한 교전
- 최종 결과: 성공
- 파견 전 예상: Phase 2 기준 예상 성공률 72%
- 실제 결과 차이: 주 목표 달성, 무사고 보너스 획득, 물리 제압 보조 조건은 일부 불안

### 주요 판정

- 충족한 필수 조건:
  - 기본 시각/청각 감지 기능 정상.
  - 소형 비살상 대응 수단 보유.
- 충족한 권장 조건:
  - Cypher 82로 출입증/감시망 오인 경보 판별 성공.
  - Cool 64로 현장 작업자 동요 억제.
  - Visibility 22로 보안 드론 위협 점수 낮게 유지.
  - `#신호대조_경험`으로 구형 출입 기록 오류를 빠르게 분리.
- 미충족 또는 불안정 조건:
  - Frame 36으로 직접 물리 제압 불리.
  - `#현장교전_경험부족`으로 난입자 접근 시 즉응 제압은 늦을 수 있음.
- 획득한 추가 베네핏:
  - 오인 경보 0회 유지.
  - 보안 드론 펌웨어 결함 기록 확보.
  - Q-Transit 출입 로그 신뢰도 상승.
- 실패한 추가 베네핏:
  - 침입자 직접 제압 보너스 미획득.
  - 침입자 장비 회수 보너스 미획득.

### 진행 로그 내부 요약

- 초반: 낡은 출입증 경고를 침입자 경보로 키우지 않고 확인 절차로 전환.
- 중반: 새벽 물류 로봇 진입 때 출입 기록 중복을 분리해 오인 경보 차단.
- 후반: 하역문 사각에서 숨어 있던 인물을 발견했으나 직접 제압은 실패.
- 마무리: 총격과 민간인 부상 없이 경비 시간 종료.

### 정산 내부 요약

- 기본 보상: 크레딧 +1,000.
- 추가 보상: 무사고 보너스 +350, 보안 드론 펌웨어 결함 기록 1개.
- 평판 변화: Q-Transit 평판 +1, 실버램프 관리자 신뢰 +1.
- 상태 태그 변화: 없음 → `#경계심`.
- 장비/임플란트 변화: 손목 연결구 정상, 소형 비살상 도구 소모 1회.
- 내부 플래그: 무허가 보안업체 펌웨어 결함 후보.

4. 숨어 있던 인물은 창고 안쪽으로 도망쳤다. 서라는 직접 몸으로 막지는 못했지만, 출구 잠금 순서를 바꿔 물품 구역 진입 전에 길을 끊었다.

5. 근무 종료 뒤 드론 로그에는 같은 배지를 두 이름으로 나눠 읽은 흔적이 남았다.

## 7. 자체 평가

### 7.1 결과가 정산 화면으로 보이는가?

그렇다.

전투 승패가 아니라 무사고 유지, 경보 처리, 로그 회수, 보상 정산 중심으로 구성했다.

### 7.2 미션/용병/매칭 예측과 연결되는가?

연결된다.

- 매칭에서 예측한 `Cypher 강점`은 오인 경보 판별로 이어졌다.
- 매칭에서 예측한 `Cool 충족`은 과잉 대응 억제로 이어졌다.
- 매칭에서 예측한 낮은 장비 노출도는 드론 위협 점수 안정으로 이어졌다.
- 매칭에서 예측한 `Frame 한계`는 직접 제압 보너스 미획득으로 이어졌다.

### 7.3 상태 태그가 부정만으로 처리되지 않는가?

그렇다.

이번 샘플은 성공 결과로 `#경계심`이라는 긍정/조건부 상태 태그를 부여했다.

### 7.4 장비/임플란트 변화와 상태 태그 변화가 분리되는가?

분리된다.

`#경계심`은 용병 현재 상태에 반영하고, 소형 비살상 도구 소모는 장비 상태에 반영한다.

### 7.5 결과 로그가 판독 가능한가?

가능하다.

플레이어는 로그만 읽어도 서라가 총을 쏘지 않고 경고를 낮췄으며, 침입자를 물품 구역 진입 전 차단했지만 직접 제압은 약했다는 점을 알 수 있다.

### 7.6 남은 검토점

- `#경계심` 같은 긍정 상태 태그의 유지/소모 규칙은 더 필요하다.
- 무사고 보너스와 직접 제압 보너스의 경제 밸런스는 더미다.
- 드론 결함 기록이 후속 미션으로 이어지는 누적 규칙은 아직 없다.
- 경비 미션 결과 로그 문체는 더 압축 가능하다.

## 8. 다음 작업 제안

다음은 결과 보고서 1차/2차 샘플 비교 검토다.

비교할 것:

1. 환경/정비 미션과 경비/오인식별 미션에서 같은 결과 보고서 구조가 유지되는가.
2. 상태 태그 변화와 장비/임플란트 변화 분리가 반복 가능한가.
3. 부정 상태 전환과 긍정 상태 부여가 같은 규격 안에서 작동하는가.
4. 결과 로그가 판정표를 대체하지 않고 플레이어 판독을 유도하는가.
