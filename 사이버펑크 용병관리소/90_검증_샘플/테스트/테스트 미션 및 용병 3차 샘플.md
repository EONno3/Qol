# 테스트 미션 및 용병 3차 샘플

이 문서는 2차 샘플 이후 만든 3차 테스트 문서다.

중요:

- 태그명, 수치, 성공 전망, 성공률은 전부 **가칭/검증용 더미 데이터**다.
- 실제 태그 사전 확정본이 아니다.
- 목적은 다른 미션 성격에서도 Phase 정보 해금, 스탯 요구 개연성, 미션 슬롯 매칭이 작동하는지 검증하는 것이다.
- 이 문서는 `기본 분석기`, `고급 분석기`, `상위 분석기` 같은 구버전 표현을 포함한다. 최신 기준에서는 **용병 분석 기관**과 **미션 분석 기관**을 분리하고, 미션 슬롯 매칭은 `min(용병 분석 레벨, 미션 분석 레벨)`을 따른다.
- 이번 샘플은 `Frame/Wire/Visibility` 중심이 아니라 `Cypher/Cool/오인식별 대응` 중심이다.

## 1. 테스트 목표

- 평범한 경비 미션에 왜 `Cypher`가 필요한지 납득되는가?
- 총을 잘 쏘는 용병이 항상 경비 미션 정답이 아님을 보여주는가?
- `Cool`이 교전 회피와 오인 대응에 의미 있게 작동하는가?
- Phase 1에서는 흐릿한 위험 신호만 보이고, Phase 2에서 보안 시스템 조건이 명확히 열리는가?
- 미션 매칭 정보가 캐릭터 창이 아니라 미션 슬롯 대입 화면에서만 뜨는가?
- 이력서가 특정 미션에 맞춰 작성된 것처럼 보이지 않는가?

## 2. 테스트 미션: 실버램프 야간 경비 대행

### 내부 설계

- 미션 타입: 경비 / 오인식별 위험 / 제한 교전
- 위치: 중층 Ring, 실버램프 임대 창고
- 관련 팩션: Q-Transit, Plus One, 무허가 보안업체
- 의뢰자: 실버램프 창고 임시 관리자
- 목표: 야간 6시간 동안 창고 구역 경비, 출입 로그 유지, 오인 교전 방지
- 위험도: 낮음처럼 보이나 실제는 중간
- 보상: 크레딧 보통, Q-Transit 평판 소폭, 보안 장비 할인 가능

### 스탯/태그 요구 개연성

- Cypher 요구 이유: 창고 보안 시스템은 저가 AI 감시망과 구형 출입증 DB를 억지로 묶어둔 상태다. 새로 등록된 아군 운송 인력도 종종 침입자로 오인된다. 용병이 현장에서 식별 신호를 읽고, 오인 경보를 낮춰야 한다.
- Cool 요구 이유: 보안 드론이 경고음을 울리면 현장 노동자들이 동시에 움직인다. 용병이 바로 사격하거나 위협하면 단순 경고가 교전 사고로 커진다.
- Visibility 요구 이유: 고화력 외장 무장은 드론의 위협 점수를 올린다. 이번 미션은 침입자를 죽이는 일이 아니라, 정상 출입자를 잘못 쏘지 않는 일이 더 중요하다.
- Pulse 요구 이유: 일부 침입자는 정상 운송 동선 사이에 끼어 들어온다. 로그상 이상은 늦게 드러나고, 현장 감각으로 이상한 흐름을 먼저 잡을 수 있다.
- Frame 요구 한계: 물리 제압은 마지막 수단이다. 높은 Frame은 난입자 제압에는 좋지만, 과잉 대응 위험을 같이 키운다.

### 필수/권장/추가 베네핏

아래 명칭은 더미다.

- 필수: 기본 시각/청각 감지 기능 정상
- 필수: 비살상 대응 수단 또는 제압 절차
- 권장: `Cypher 60 이상`
- 권장: `Cool 55 이상`
- 권장: `Visibility 50 이하`
- 보조: `Pulse 50 이상`
- 위험: `#과잉대응_성향`, `#트리거해피`, `Visibility 75 이상`, `#기업DB_수배흔적`
- 추가 베네핏: 오인 경보 0회 유지
- 추가 베네핏: 숨은 침입자 조기 발견
- 추가 베네핏: Q-Transit 출입 로그 신뢰도 상승
- 추가 베네핏: 보안 드론 펌웨어 결함 기록 확보

## 3. 미션 카드

### Phase 0: 초기 화면

> **의뢰명:** 실버램프 야간 경비 대행  
> **위치:** 중층 Ring, 임대 창고 구역  
> **보상:** 크레딧 보통, 무사고 보너스 가능

#### 미션 브리핑

실버램프 창고 관리자가 하룻밤짜리 경비 인력을 찾고 있다. 물건은 비싸지 않지만, 최근 구역 출입 기록이 자꾸 꼬이고 있다. 경비 드론은 낡은 출입증을 보면 경고음을 울리고, 작업자들은 “괜히 눈 마주치면 총 맞는 거 아니냐”고 불평한다. 관리자는 총격 사건 없이 조용히 밤을 넘기는 걸 가장 중요하게 여긴다. 창고 뒤쪽 하역문은 카메라 사각이 조금 있고, 새벽 두 시마다 운송 로봇이 몰려 들어와 현장이 복잡해진다.

- 🧠 **[요구 스탯]** `Cypher 60 이상` 권장: 출입증/감시망 오인 경보 판별
- 🧊 **[요구 스탯]** `Cool 55 이상` 권장: 경고 상황에서 과잉 대응 억제

- ⚡ **[보조 스탯]** `Pulse 50 이상`: 하역 동선 이상 감지
- ⚠️ **[위험 조건]** `#과잉대응_성향`, `#트리거해피`, `Visibility 75 이상`, `#기업DB_수배흔적`

## 8. 3차 자체 평가

### 8.1 스탯 요구 개연성

경비 미션인데도 `Cypher`가 필요한 이유를 오인 경보와 출입증 DB 문제로 설명했다.

`Cool`은 적을 죽이는 능력이 아니라, 경고 상황을 교전으로 키우지 않는 능력으로 작동한다.

### 8.2 선택지 분화

- 도겸: 전투력은 좋지만 미션 목표와 충돌.
- 서라: 미션 핵심은 가장 잘 풀지만 물리 제압 약함.
- 한비오: 균형형. 감각/침착은 좋고, 보안망 판별은 경계값.

### 8.3 Phase 정보 해금

- Phase 0: 경비 드론, 작업자 불안, 출입 기록 꼬임으로 암시.
- Phase 1: 성공 전망과 대표 매칭 신호만 표시.
- Phase 2: `Cypher`, `Cool`, `Visibility`, 위험 태그, 정확한 예상 성공률 공개.

### 8.4 이력서-미션 분리

- 이력서에는 특정 미션명, 목표, 요구 조건을 직접 풀어 쓰지 않는다.
- 이력서에는 용병의 상시 경력, 습관, 장비, 상태만 둔다.
- 미션 요구와 용병 정보를 연결하는 해석은 미션 슬롯 매칭 테스트에만 둔다.

### 8.5 2차 샘플과 다른 검증점

2차는 환경/정비/검문 미션이었다.

3차는 경비/오인식별/제한 교전 미션이다.

같은 카드 구조가 다른 미션 성격에도 적용되는지 보는 용도다.

### 8.6 남은 검토점

- `Cypher`와 `Wire`의 역할 경계가 더 필요하다.
- `Cool`이 교전 억제 외에 협상/공포/패닉에 어떻게 쓰이는지 샘플 더 필요하다.
- `Visibility`는 수치 후보로 유효하지만 산식은 아직 없다.
- 성공 전망/성공률은 아직 더미다.
