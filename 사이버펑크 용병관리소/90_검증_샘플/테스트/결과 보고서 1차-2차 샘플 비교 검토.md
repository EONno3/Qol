# 결과 보고서 1차-2차 샘플 비교 검토

이 문서는 `테스트 결과 보고서 1차 샘플.md`와 `테스트 결과 보고서 2차 샘플.md`를 비교해, 결과 보고서 제작 규칙으로 유지할 항목과 아직 미정인 항목을 정리한다.

중요:

- 두 샘플의 태그명, 수치, 보상, 성공률은 더미다.
- 목적은 콘텐츠 확정이 아니라 파견 후 결과 보고서 구조 검증이다.
- 결과 보고서는 플레이어가 사건 로그를 읽고 다음 판단을 하는 텍스트 판독 화면이다.

## 1. 샘플 역할

### 1차 결과 보고서

- 기준 미션: `7번 바이패스 밸브 차단`
- 투입 용병: 민도하 / 누수음
- 결과 유형: 부분 성공
- 검증 축: 환경 위험, 정비, 기록 장치 회수, 장비/임플란트 악화, 부정 상태 태그 전환
- 핵심 질문: 기술적 성공과 물리/환경 손실이 함께 발생할 때 보고서가 납득되는가?

### 2차 결과 보고서

- 기준 미션: `실버램프 야간 경비 대행`
- 투입 용병: 서라 / 배지
- 결과 유형: 성공
- 검증 축: 오인식별, 무사고 유지, 제한 교전, 긍정/조건부 상태 태그 부여
- 핵심 질문: 전투가 적은 경비 미션도 사건 로그와 정산으로 재미있게 읽히는가?

## 2. 공통으로 작동한 구조

### 2.1 결과 보고서 기본 흐름

작동한 점:

- 두 샘플 모두 `요약 → 진행 로그 → 판정/신호 → 정산 → 다음 화면 반영` 흐름을 가진다.
- 진행 로그가 먼저 나오고, 태그/스탯/권장 기준은 뒤의 분석 영역에서 확인된다.
- 플레이어는 로그를 먼저 읽고, 분석 정보로 자신이 읽은 원인을 확인할 수 있다.

유지할 규칙:

- 플레이어용 진행 로그에는 태그명, 스탯 수치, 권장 기준을 직접 섞지 않는다.
- Phase 1/2에서도 사건 로그를 유지한다.
- 분석 영역은 로그를 대체하지 않는다.

### 2.2 미션/용병/매칭과의 연결

작동한 점:

- 1차는 민도하의 정비/설비음/피로/호흡 보조 문제가 결과로 이어졌다.
- 2차는 서라의 출입 기록 판독, 낮은 장비 노출, 교전 경험 부족이 결과로 이어졌다.
- 미션 슬롯 매칭에서 예측한 장점과 위험이 보고서에서 회수된다.

유지할 규칙:

- 결과 보고서는 미션 카드나 이력서에 없던 핵심 위험을 갑자기 정답처럼 꺼내면 안 된다.
- 매칭 예측은 결과 보고서의 판정 근거가 될 수 있다.
- 결과 보고서는 이력서 내용을 새로 꾸며 넣는 곳이 아니라, 파견에서 실제 발생한 일을 기록하는 곳이다.

### 2.3 상태 태그 변화

작동한 점:

- 1차는 `#피로누적 → #심각한_피로누적`으로 부정 상태 태그 전환을 검증했다.
- 2차는 `없음 → #경계심`으로 긍정/조건부 상태 태그 부여를 검증했다.
- 상태 태그가 스탯처럼 수치 증가하지 않고, 태그 추가/전환으로 처리된다.

유지할 규칙:

- 상태는 스탯이 아니라 상태 태그다.
- 상태 변화는 `신규 부여`, `악화`, `완화`, `해소`, `상위 전환`, `하위 전환`, `소모`, `병합`, `변질`로 표현한다.
- 상태 태그명은 플레이어가 효과를 직관적으로 추론할 수 있어야 한다.
- 긍정 상태 태그도 미션 결과로 생기거나 소모될 수 있다.

### 2.4 장비/임플란트 변화

작동한 점:

- 1차는 `#호흡보조_불안정 → #호흡보조_기능저하`를 장비/임플란트 변화로 분리했다.
- 2차는 소형 비살상 도구 1회 소모를 장비 소모로 처리했다.
- 신체 상태, 상태 태그, 장비/임플란트 변화가 분리되었다.

유지할 규칙:

- 임플란트 기능 저하를 신체 상태 태그로 대체하지 않는다.
- 장비 소모, 필터 오염, 기능 저하는 장비/임플란트 변화로 우선 처리한다.
- 그로 인한 숨참, 기침, 떨림 같은 증상은 로그나 부가 증상 설명으로 둔다.

### 2.5 보상, 손실, 추가 베네핏 미획득

작동한 점:

- 보상은 수치로 표시된다.
- 실제 손실과 얻지 못한 보너스가 분리되기 시작했다.
- 2차에서 `침입자 직접 제압 보너스 없음`, `침입자 장비 회수 보너스 없음`이 손실이 아니라 추가 베네핏 미획득으로 분리되었다.

유지할 규칙:

- 플레이어가 가진 자원/상태/장비가 깎이면 손실이다.
- 얻을 수 있었지만 얻지 못한 항목은 추가 베네핏 미획득이다.
- 더미 수치라도 보고서 본문에서는 실제 정산처럼 표기한다.

### 2.6 후속 단서와 내부 플래그

작동한 점:

- 플레이어용 후속 단서와 내부 플래그가 분리되었다.
- 2차는 Phase에 따라 `드론 로그 조각`, `보안 드론 결함 의심 로그`, `보안 드론 펌웨어 결함 기록`으로 해석 깊이를 조절했다.

유지할 규칙:

- 후속 단서는 너무 노골적인 정답명으로 쓰지 않는다.
- Phase가 오르면 같은 회수물이 더 정확히 해석될 수 있다.
- `경계 후보`, `후속 미션 플래그`, `이벤트 후보` 같은 내부값은 플레이어 보고서 본문에 직접 쓰지 않는다.

## 3. Phase별 결과 보고서 구조

### Phase 0

작동한 점:

- 발생 사실, 보상, 손실, 상태 태그 변화, 장비 변화는 표시된다.
- 정확한 숨김 태그명, 계산식, 조건 충돌명은 숨긴다.
- 사건 로그는 텍스트 판독용으로 남는다.

유지할 규칙:

- 플레이어가 만든 결과는 Phase 0에서도 숨기지 않는다.
- 단, 원인 해석은 현상 중심으로 둔다.

### Phase 1

작동한 점:

- Phase 0 로그를 유지하면서 대표 원인 신호를 추가한다.
- 정확한 내부 계산식은 숨긴다.
- 긍정/부정/조건부 신호가 함께 표시될 수 있다.

유지할 규칙:

- Phase 1은 정답표가 아니라 신호 표시다.
- 신호는 플레이어가 로그에서 읽은 가설을 확인하는 정도여야 한다.

### Phase 2

작동한 점:

- 정확한 조건 충돌, 스탯 판정, 태그/장비 속성 반응을 공개한다.
- 진행 로그에는 여전히 태그명/스탯 수치를 직접 섞지 않는다.
- 정확한 판정 영역에서만 태그명과 수치를 사용한다.

유지할 규칙:

- Phase 2라고 해서 사건 로그를 판정표로 대체하지 않는다.
- Phase 2에서도 이야기 핵심 비밀은 일부 숨길 수 있다.

## 4. 반복된 위험과 방지 규칙

### 4.1 상태를 스탯처럼 쓰는 문제

문제:

- `피로 +1단계`, `피로 증가`처럼 상태를 수치 게이지처럼 처리할 위험이 있었다.

방지 규칙:

- 상태는 상태 태그로 처리한다.
- 피로 계열 변화는 `#피로누적 → #심각한_피로누적`처럼 태그 전환으로 쓴다.

### 4.2 장비/임플란트 문제를 신체 상태로 뭉개는 문제

문제:

- 호흡 보조 장비 문제를 `#호흡기_자극` 같은 상태 태그로 뭉갤 위험이 있었다.

방지 규칙:

- 임플란트/장비 기능 저하는 장비/임플란트 변화로 처리한다.
- 신체 증상은 로그나 증상 설명으로 분리한다.

### 4.3 손실과 미획득 혼동

문제:

- 침입자 장비 회수 실패처럼 얻지 못한 보너스를 손실로 적을 위험이 있었다.

방지 규칙:

- 이미 가진 것이 줄면 손실이다.
- 얻을 수 있던 추가 보상이 사라지면 추가 베네핏 미획득이다.

### 4.4 상태 태그명 직관성

문제:

- `#근무리듬_안정`처럼 효과가 바로 읽히지 않는 상태 태그가 생길 수 있었다.

방지 규칙:

- 상태 태그명은 플레이어가 다음 선택에서 어떤 의미인지 바로 떠올릴 수 있어야 한다.
- `#경계심`처럼 미션 효과와 연결되는 이름을 우선한다.

## 5. 아직 미정인 것

- 상태 태그의 유지/소모 규칙.
- 긍정 상태 태그가 연속으로 누적될 수 있는지 여부.
- 장비 소모의 정비비/충전비 산식.
- 보상, 무사고 보너스, 정비비의 경제 밸런스.
- 후속 단서 누적 방식.
- Phase 1 신호 기호의 정확한 의미.
- 결과 보고서 문체 압축 기준.

## 6. 다음 단계 판단

지금 바로 결과 보고서 샘플을 더 늘리는 것보다, 두 샘플에서 나온 규칙을 문서에 반영하는 것이 우선이다.

이유:

- 환경/정비형 결과와 경비/오인식별형 결과가 모두 생겼다.
- 부정 상태 전환과 긍정 상태 부여가 모두 검증되었다.
- 장비/임플란트 변화와 상태 태그 변화 분리 규칙이 반복 확인되었다.

다음 작업:

1. `결과 보고서 제작 규격.md`에 비교 검토 결과 중 부족한 규칙을 보강한다.
2. `프로젝트 온톨로지.md`에 결과 보고서 최신 구조를 반영한다.
3. 이후 상태 태그/장비 변화/후속 단서 후보군을 소규모로 정리한다.

## 7. 현재 결론

결과 보고서 1차/2차 샘플은 파견 후 정보 구조 검증용 기준 후보로 유지한다.

다만 최종 콘텐츠 예시는 아니다.

다음 핵심은 결과 보고서 규격과 온톨로지를 최신 샘플 기준으로 정리하는 것이다.
