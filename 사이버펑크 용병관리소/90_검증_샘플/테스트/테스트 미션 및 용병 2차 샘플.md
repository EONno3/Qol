# 테스트 미션 및 용병 2차 샘플

이 문서는 1차 샘플 검토 후 수정한 2차 테스트 문서다.

중요:

- 태그명, 수치, 성공률은 전부 **가칭/검증용 더미 데이터**다.
- 실제 태그 사전 확정본이 아니다.
- 목적은 미션 카드, 용병 이력서, 매칭 화면, Phase 정보 해금 구조 검증이다.
- 이 문서는 `기본 분석기`, `고급 분석기`, `상위 분석기` 같은 구버전 표현을 포함한다. 최신 기준에서는 **용병 분석 기관**과 **미션 분석 기관**을 분리하고, 미션 슬롯 매칭은 `min(용병 분석 레벨, 미션 분석 레벨)`을 따른다.
- 1차 샘플에서 잘못 들어간 `캐릭터 창의 미션 적성표`, `노드 예측`, `정의 없는 보험 리스크`, `단서 목록식 이야기 해금`은 제거했다.

## 1. 1차 검토 반영 사항

- 스탯 요구에는 미션 내부 개연성을 붙인다.
- 미션 카드에서 `노드 예측`은 빼고, 대신 `추가 베네핏`을 둔다.
- `파손/보험 리스크`는 산식 미정이므로 용병 카드에서 제거한다.
- 이력서 문장은 특정 미션에 맞춘 표현이 아니라 인물 성향으로 쓴다.
- 캐릭터 Phase 2에는 미션 매칭 요약을 넣지 않는다.
- 캐릭터 Phase 2는 상세 태그, 숨겨진 부정/만성/태생 태그, 확장 이력을 보여준다.
- 매칭 신호는 캐릭터 창이 아니라 **미션 슬롯에 용병을 대입했을 때** 표시한다.
- 이야기 정보는 `공개 이야기 단서` 목록이 아니라 이력 문장 자체가 확장되는 방식으로 연다.
- 예상 성공은 Phase 1에서 거친 전망으로, Phase 2에서 정확한 수치로 표시한다.
- 장비 가시성은 긍정/부정 태그가 아니라 장비 속성 또는 수치 후보로 분리한다.
- 이력서는 특정 미션에 맞춘 해석을 쓰지 않고, 상시 고용 정보로만 작성한다.

## 2. 테스트 미션: 7번 바이패스 밸브 차단

### 내부 설계

- 미션 타입: 지원 / 환경 위험 / 제한적 검문
- 위치: 하층 Pipe, The Valve 관할 폐수 우회로
- 관련 팩션: The Valve, 알-마디낫 하청 보수 라인
- 의뢰자: The Valve 현장 감독 대리
- 목표: 굳은 바이패스 밸브 수동 차단, 압력 기록 장치 회수
- 위험도: 중간 이상
- 보상: 크레딧 보통, 플럭스 소량, The Valve 평판 소폭

### 스탯/태그 요구 개연성

- Frame 요구 이유: 밸브 원래 자동 구동 모터가 돌리는 설비다. 침수와 부식으로 모터가 죽었고, 현장에는 비상 수동 토크 핸들만 남았다. 핸들은 오래 굳어 일반 작업자 둘이 매달려도 거의 움직이지 않는다.
- Wire 요구 이유: 압력 기록 장치는 구형 아날로그 센서와 저가 디지털 저장부가 섞인 장비다. 전원을 끄면 기록이 날아가므로, 현장에서 직접 선을 우회하고 저장부를 분리해야 한다.
- 낮은 Visibility 요구 이유: 입구 임시 검문대는 알-마디낫 보안 스캐너를 빌려 쓰고 있다. 거대한 외장 무기나 노출된 대형 크롬은 작업 장비가 아니라 무장 침입으로 분류될 수 있다.
- 호흡/환경 대응 요구 이유: 폐수 우회로에 황색 부식성 가스가 고여 있다. 짧은 진입은 가능하지만 필터나 보조 호흡 없이는 작업 시간이 급격히 줄어든다.
- 협소 지형 대응 이유: 밸브실로 가는 정비 터널은 장비를 몸에 붙이고 지나가야 할 정도로 좁다. 큰 외장 장비, 폐소 관련 만성 상태, 낮은 Cool은 위험을 키운다.

### 필수/권장/추가 베네핏

아래 명칭은 더미다.

- 필수: `#호흡_필터` 또는 `#방독_대응`
- 필수: 기본 이동 파츠 정상
- 권장: `Frame 60 이상`
- 권장: `Visibility 40 이하` 또는 `#내장형_도구`
- 권장: `Wire 55 이상`
- 위험: `Visibility 70 이상`, `#폐소공포증`, `#호흡계_마모`
- 추가 베네핏: 압력 기록 장치 무손상 회수
- 추가 베네핏: The Valve 현장 작업자 호감 확보
- 추가 베네핏: 잉여 플럭스 소량 회수
- 추가 베네핏: 알-마디낫 하청 은폐 기록 일부 발견
- 추가 베네핏: 장비 오염 감소

## 3. 미션 카드

### Phase 0: 초기 화면

> **의뢰명:** 7번 바이패스 밸브 차단  
> **위치:** 하층 Pipe, The Valve 폐수 우회로  
> **보상:** 크레딧 보통, 현장 보너스 가능

#### 미션 브리핑

7번 우회로의 자동 밸브 구동부가 침수로 멈췄다. 현장 감독은 “비상 수동 핸들이 아직 살아 있다”고 말했지만, 먼저 들어간 작업자 둘은 핸들을 반 바퀴도 못 돌리고 나왔다. 밸브실로 가는 정비 터널은 성인 한 명이 장비를 끌어안고 겨우 지나갈 폭이고, 내부 산소통 안쪽은 누렇게 변색되어 있었다. 입구 임시 검문대는 작업자 가방까지 열어보고 있으며, 눈에 띄는 무장과 대형 크롬을 작업 장비로 봐줄 분위기는 아니다.

### Phase 1: 기본 분석기 해금

미션 단독 화면에서는 Phase 0 브리핑 유지.

매칭 정보는 표시하지 않는다. 매칭 정보는 용병을 미션 슬롯에 올렸을 때만 뜬다.

### Phase 2: 고급 분석기 해금

> **의뢰명:** 7번 바이패스 밸브 차단  
> **미션 타입:** 지원 / 환경 위험 / 제한적 검문

#### 시스템 요구 사항

- 🛡️ **[필수 장비]** `#호흡_필터` 또는 `#방독_대응` 보유자
- 🦿 **[필수 상태]** 기본 이동 파츠 정상
- 💪 **[요구 스탯]** `Frame 60 이상` 권장: 비상 수동 토크 핸들 개폐
- 🧰 **[요구 스탯]** `Wire 55 이상` 권장: 압력 기록 장치 우회 분리
- 👤 **[권장 장비 속성]** `Visibility 40 이하` 또는 `#내장형_도구`
- ⚠️ **[위험 조건]** `Visibility 70 이상`, `#폐소공포증`, `#호흡계_마모`

#### 가시성 처리 기준

- `Visibility`는 장비 속성/수치 후보다.
- `#산업용_크롬`, `#외장형_장비`, `#내장형_도구`는 장비 성격 태그다.
- 가시성이 높은 장비가 항상 부정인 것은 아니다.
- 검문/잠입/기업 미션처럼 노출을 싫어하는 미션에서만 `Visibility`가 위험값으로 해석된다.

#### 추가 베네핏

- 압력 기록 장치 무손상 회수 시 추가 보상
- The Valve 현장 작업자 호감 확보 시 후속 의뢰 할인
- 잉여 플럭스 회수 가능
- 알-마디낫 하청 은폐 기록 발견 가능
- 오염 대응 우수 시 장비 정비비 감소

Phase 0 브리핑 유지.

## 4. 테스트 용병 1: 강석도

## 8. 2차 자체 평가

### 8.1 스탯 요구 개연성

이제 Frame, Wire, 낮은 Visibility, 호흡 대응이 왜 필요한지 미션 안에서 설명된다.

### 8.2 정보 위치

캐릭터 창에는 캐릭터 정보만 있다.

미션-용병 매칭 정보는 미션 슬롯 대입 화면에만 있다.

### 8.3 캐릭터 Phase 2

캐릭터 Phase 2는 `보유 태그 상세 내역`과 확장 이력 중심이다.

미션 추천표는 없다.

### 8.4 이야기 해금

이야기 정보는 `공개 단서` 목록이 아니라 이력 문장 확장으로 처리했다.

### 8.5 이력서-미션 분리

- 이력서에는 특정 미션명, 목표, 장치 요구를 직접 풀어 쓰지 않는다.
- 이력서에는 용병의 상시 경력, 습관, 장비, 상태만 둔다.
- 미션 요구와 용병 정보를 연결하는 해석은 미션 슬롯 매칭 테스트에만 둔다.

### 8.6 남은 검토점

- Phase 1 성공 전망은 등급/범위 더미다.
- Phase 2 성공률 숫자 산식은 아직 없다. 지금은 UI 감각 확인용 더미다.
- 태그명은 확정 전이다.
- `추가 베네핏`이 보상/후속 의뢰/장비비 감소 중 어디까지 시스템화될지 결정 필요.
- Phase 1 매칭 기호 규칙은 별도 문서로 더 고정해야 한다.
