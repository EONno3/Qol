# 테스트 결과 보고서 1차 샘플

이 문서는 결과 보고서 제작 규격을 검증하기 위한 1차 샘플이다.

중요:

- 태그명, 수치, 보상, 성공률은 전부 **가칭/검증용 더미 데이터**다.
- 실제 태그 사전 확정본이 아니다.
- 목적은 파견 후 결과가 용병 이력서, 장비 상태, 평판, 후속 단서로 어떻게 되돌아가는지 검증하는 것이다.
- 기준 미션은 `테스트 미션 및 용병 2차 샘플.md`의 `7번 바이패스 밸브 차단`이다.

## 1. 테스트 목표

- 결과 보고서가 추론 화면이 아니라 정산 화면으로 보이는가?
- 성공/부분 성공/실패 이유가 납득되는가?
- 상태 태그 변화와 파손/오염이 숨김 없이 표시되는가?
- 미션 슬롯 매칭에서 예측한 위험과 실제 결과가 연결되는가?
- 결과가 다음 화면에 어떻게 반영되는지 명확한가?

## 2. 내부 결과 설계

### 기본 정보

- 미션명: 7번 바이패스 밸브 차단
- 투입 용병: 민도하 / 누수음
- 미션 타입: 지원 / 환경 위험 / 제한적 검문
- 최종 결과: 부분 성공
- 파견 전 예상: Phase 2 기준 예상 성공률 64%
- 실제 결과 차이: 주 목표 달성, 추가 베네핏 일부 실패, 상태 태그 변화 발생

### 주요 판정

- 충족한 필수 조건:
  - 기본 이동 파츠 정상.
  - 호흡 보조 장비 보유.
- 미충족 또는 불안정 조건:
  - 호흡 보조 장비 안정성 부족.
  - Frame 권장 기준 미달.
- 충족한 권장 조건:
  - Wire 76으로 기록 장치 우회 분리 성공.
  - Visibility 40 이하 기준 충족.
  - 현장 정비 경험과 설비음 판별로 작업 지연을 일부 줄임.
- 미충족한 권장 조건:
  - Frame 60 이상 권장 미충족.
  - 출격 전 호흡 보조 정비 없음.
- 획득한 추가 베네핏:
  - 압력 기록 장치 일부 회수.
  - 알-마디낫 하청 은폐 기록 조각 발견.
  - The Valve 현장 작업자 호감 소폭 상승.
- 실패한 추가 베네핏:
  - 압력 기록 장치 무손상 회수 실패.
  - 잉여 플럭스 회수 실패.
  - 장비 오염 감소 실패.

### 진행 로그 내부 요약

- 진입: 낮은 Visibility와 현장 작업자 안면으로 임시 검문 통과.
- 이동: 독성 공기 노출. 호흡 보조 장비가 버텼지만 출력이 불안정.
- 주요 작업: Wire 판정 성공으로 압력 기록 장치 저장부 일부 보존.
- 물리 작업: Frame 권장 기준 미달로 수동 토크 핸들 작업 시간이 증가.
- 돌발 노드: 작업 시간이 늘면서 호흡계 상태 악화.
- 조커 카드: `#설비음_판별`이 압력 역류를 조기 감지해 탈출 지연을 막음.

### 정산 내부 요약

- 기본 보상: 크레딧 +1,200.
- 추가 보상: 플럭스 +0, 젖은 정산표 조각 1개.
- 평판 변화: The Valve 평판 +1.
- 플레이어용 후속 단서: 하청 코드가 찍힌 손상 로그.
- 내부 플래그: 알-마디낫 하청 라인 경계 후보.
- 상태 태그 변화: `#피로누적` → `#심각한_피로누적`.
- 장비/임플란트 변화: 정밀 손가락 툴 오염, `#호흡보조_불안정` → `#호흡보조_기능저하`.
- 후속 단서: 하청 코드가 찍힌 손상 로그.

> **결과:** 부분 성공

1. 민도하는 작업자 통로로 진입했다. 장비 노출은 낮았고, 현장 작업자 몇 명이 그를 알아봐 검문은 길어지지 않았다.

4. 수동 토크 핸들은 생각보다 더 굳어 있었다. 혼자 힘으로 돌리는 데 시간이 걸렸고, 밸브실 내부 압력이 한 차례 튀었다.
5. 귀 뒤쪽 청각 보조 장치가 압력 역류음을 먼저 잡았다. 덕분에 탈출로가 막히기 전에 밸브실을 빠져나왔다.

1. 민도하는 작업자 통로로 진입했다. 장비는 작업복 안쪽에 붙어 있었고, 현장 작업자 몇 명이 그를 알아봐 임시 검문은 길어지지 않았다.

- 젖은 정산표 조각.

## 6. 다음 화면 반영

### 민도하 이력서 현재 상태

결과 보고서 이후 즉시 반영:

```text
현재 상태:
- #심각한_피로누적
```

숨기지 않는 이유:

- 이번 미션에서 플레이어가 직접 만든 상태 태그 변화다.
- 정보 해금 Phase와 무관하게 즉시 공개한다.

### 장비/임플란트 상태

```text
정밀 손가락 툴: 오염 / 세척 필요
호흡 보조 장비: 기능 저하 / 필터 교체 필요
청각 보조 장치: 정상
```

### 팩션/후속 정보

```text
The Valve: +1
하청 코드가 찍힌 손상 로그: 1개
젖은 정산표 조각: 1개
```

### 내부 반영

아래 항목은 플레이어 보고서 본문에 직접 표시하지 않는 시스템 내부 값이다.

```text
알-마디낫 하청 라인: 경계 후보
```

## 7. 자체 평가

### 7.1 결과가 정산 화면으로 보이는가?

그렇다.

미션 브리핑이 아니라 실제 발생한 결과, 보상, 손실, 상태 변화 중심으로 구성했다.

### 7.2 매칭 예측과 연결되는가?

연결된다.

- 매칭에서 예측한 `Wire 강점`은 기록 장치 일부 회수로 이어졌다.
- 매칭에서 예측한 `호흡 보조 불안정`은 `#호흡보조_기능저하`로 이어졌다.
- 매칭에서 예측한 `Frame 권장 미달`은 작업 시간 증가로 이어졌다.

### 7.3 상태 태그 변화가 숨겨지지 않는가?

숨기지 않는다.

`#심각한_피로누적`을 즉시 표시한다.

임플란트/장비 쪽 변화인 `#호흡보조_기능저하`와 필터 교체 필요는 장비/임플란트 상태에 따로 표시한다.

### 7.4 다음 선택으로 이어지는가?

이어진다.

- 민도하를 바로 다시 보낼지 휴식/치료할지 판단하게 만든다.
- 호흡 보조 필터 교체나 장비 세척 비용이 생긴다.
- 하청 코드가 찍힌 손상 로그와 젖은 정산표 조각이 후속 단서가 된다.

### 7.5 Phase별 보고서로 보이는가?

보인다.

- Phase 0은 발생 사실과 정산 중심이다.
- Phase 1은 원인 신호를 추가한다.
- Phase 2는 정확한 조건 충돌과 태그/스탯 판정을 공개한다.

### 7.6 남은 검토점

- 정비비와 치료비 수치는 더미다.
- 평판 상승/경계 후보의 정확한 누적 규칙이 없다.
- 후속 단서 누적 방식이 아직 없다.
- 결과 로그 문체는 더 압축 가능하다.
