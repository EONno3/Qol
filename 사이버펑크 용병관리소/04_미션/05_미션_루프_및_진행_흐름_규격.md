# 미션 루프 및 진행 흐름 규격

이 문서는 픽서 게시판의 미션 수주부터 분석 정보 해금, 용병 대입 매칭, 파견 작전 수행, 미션 실패 시의 구역별 생존 판정, 최종 결과 정산까지 이어지는 **게임플레이 루프(Core Gameplay Loop)**와 **구역별 상세 진행 흐름**을 정의하는 표준 기획 규격서이다.

---

## 0. MVP 적용 범위

이 문서는 장기 설계 전체를 포함하지만, 실제 제작은 **MVP 필수 기능**부터 진행한다.

### 0.1 MVP 필수

MVP에서 반드시 작동해야 하는 최소 범위다.

- 미션 게시판에 소수의 의뢰가 생성된다.
- 미션 카드는 Phase 0~2 구조를 가지되, 우선 수동 데이터로 작성한다.
- 용병 이력서는 Phase 0~2 구조를 가지되, 우선 수동 데이터로 작성한다.
- 용병 분석 기관과 미션 분석 기관은 `Lv.0~Lv.2`만 사용한다.
- 미션 슬롯 매칭은 `min(용병 분석 레벨, 미션 분석 레벨)` 규칙을 따른다.
- 기본 노드 판정, 실패/성공, 보상, 상태 변화, 결과 보고서가 한 번의 루프로 이어진다.

### 0.2 고도화 후보

MVP 이후 확장할 수 있는 항목이다.

- 액티브 분석 슬롯의 대기 시간, 연산 비용, 슬롯 개수 확장.
- 미션 분석 슬롯에 들어간 미션의 방치 시간 보존.
- 구역별 생존/사망 판정 수식의 세부 수치 밸런싱.
- 로컬 AI(Gemma 계열)를 통한 브리핑 자동 생성.
- 구역/팩션별 미션 생성 가중치와 고급 의뢰 테이블.

### 0.3 보류

현재는 설계만 남기고 구현 대상으로 보지 않는다.

- 전체 UI 세부 연출.
- 모든 태그/장비/임플란트 대량 사전화.
- 모든 위험 노드의 확률 계산식 확정.
- 최종 문체 확정.
- 경제 밸런스 전체 수치 확정.

## 1. 미션 코어 루프 사이클 (Core Gameplay Loop)

게임의 하루는 픽서 게시판의 의뢰를 관리하고 용병단을 정비하는 루프로 순환한다.

```text
[1. 의뢰 탐색 및 방치] ➔ [2. 용병/미션 이중 분석 기관 판독] ➔ [3. 매칭 슬롯 교차 연산]
       ➔ [4. 작전 수행 (노드 판정)] ➔ [5. 생존 판정 및 결과 정산]
```

### 1.1 의뢰 탐색 및 방치 시간 (Decay Time) 규칙
*   **미션 생성**: 매일 아침(턴 시작 시) 픽서 네트워크를 통해 하층, 중층, 상층의 신규 의뢰 카드가 게시판에 일정 개수 무작위 생성된다.
*   **방치 시간 (Decay)**: 모든 미션 카드에는 `방치 시간 (예: 2일, 3일)` 상수가 존재한다.
    *   하루(1턴)가 경과할 때마다 방치 시간이 1일씩 감소한다.
    *   방치 시간이 `0`이 되면 해당 의뢰는 픽서 네트워크에서 **영구 증발**하며, 플레이어가 수락했던 계약일 경우 일정 수준의 **평판 패널티**가 부과된다.
    *   상층 미션일수록 팩션들의 정치적 기밀성이 높아 방치 시간이 극도로 짧다 (통상 1일~2일).

### 1.2 스테이션 이중 분석 기관 및 연산 슬롯 규칙
스테이션의 정보 해금은 하나의 만능 분석 데크가 아니라, **용병 분석 기관**과 **미션 분석 기관**으로 분리된다.

두 기관은 서로 다른 대상을 판독한다.

*   **용병 분석 기관**: 용병 이력서, 용병의 상태/태그/장비 판독 깊이를 결정한다.
*   **미션 분석 기관**: 미션 카드/요약서, 미션의 요구 조건/위협/보상 판독 깊이를 결정한다.
*   **미션 슬롯 매칭**: 두 정보를 연결하는 별도 화면이며, 항상 `용병 분석 레벨`과 `미션 분석 레벨`을 함께 참조한다.

#### A. 분석 레벨과 페이지 해금
각 분석 기관은 `Lv.0~Lv.2`의 정보 깊이를 가진다. 숫자는 해당 문서에서 열리는 페이지/Phase와 대응한다.

| 분석 레벨 | 용병 분석 기관: 용병 이력서 | 미션 분석 기관: 미션 카드/요약서 |
| :---: | :--- | :--- |
| **Lv.0** | Phase 0: 이름, 출신, 이력 단서, 현재 상태 중심 | Phase 0: 의뢰명, 위치, 보상, 철수 패널티, 브리핑 중심 |
| **Lv.1** | Phase 1: 5대 스탯 요약, 대표 분석, 장비/임플란트 요약 | Phase 1: 추천 스탯 체급, 장비 카테고리 단서, 위험 방향 신호 |
| **Lv.2** | Phase 2: 시스템 태그 상세, 만성 상태, 장비/임플란트 상세 | Phase 2: 요구 수치, 필수 기어, 보너스 조건, 위협 프로필 요약 |

#### B. 매칭 화면 교차 해금 규칙
미션 슬롯 매칭 화면은 용병 이력서와 미션 카드를 연결하는 해석 화면이다. 따라서 한쪽 분석만 높아서는 매칭 정보가 열리지 않는다.

```text
매칭 표시 레벨 = min(용병 분석 레벨, 미션 분석 레벨)
```

*   **매칭 Lv.0**: 용병이나 미션 중 하나라도 Lv.0이면 매칭 신호를 표시하지 않는다. 용병 이력서 또는 미션 카드 자체의 독립 정보만 읽을 수 있다.
*   **매칭 Lv.1**: 두 분석 기관이 모두 Lv.1 이상이면 `유리 / 불리 / 조건부 / 미확인` 같은 방향성 신호와 거친 성공 전망을 표시한다.
*   **매칭 Lv.2**: 두 분석 기관이 모두 Lv.2 이상이면 충돌/시너지 이유, 예상 성공률, 예상 정산 범위, 추가 베네핏 가능성을 표시한다.

예:

```text
용병 분석 Lv.1 / 미션 분석 Lv.0
→ 용병 이력서 Phase 1은 열리지만, 미션 슬롯 매칭 신호는 표시하지 않는다.

용병 분석 Lv.1 / 미션 분석 Lv.1
→ 용병 이력서 Phase 1, 미션 카드 Phase 1, 매칭 Lv.1 신호가 동시에 표시된다.

용병 분석 Lv.2 / 미션 분석 Lv.1
→ 용병 상세 정보는 Phase 2까지 보이지만, 매칭 화면은 Lv.1까지만 표시된다.
```

#### C. 액티브 분석 연산 슬롯
액티브 연산 슬롯은 부족한 쪽의 분석 레벨을 특정 대상에 한해 임시로 끌어올리는 스테이션 기능이다.

*   **용병 분석 슬롯**: 특정 용병 이력서를 슬롯에 넣어 다음 단계 정보를 임시 해금한다.
*   **미션 분석 슬롯**: 특정 미션 카드를 슬롯에 넣어 다음 단계 정보를 임시 해금한다.
*   **연산 비용**: 크레딧, 정보 기록(Data), 서버 전력, 분석 인력 시간 중 일부를 소모한다.
*   **대기 시간**: 기본 1턴(하루). 다음 턴 시작 시 해당 대상의 분석 레벨을 1단계 상승시킨다.
*   **방치 시간 보존**: 미션 분석 슬롯에 들어간 미션 카드는 분석 중인 동안 방치 시간(Decay Time) 감소가 일시 정지된다.

### 1.3 용병 슬롯 매칭 연산
*   미션 카드 자체는 고정되어 있으며, 용병 이력서도 특정 미션을 전제로 작성되지 않는다.
*   매칭 슬롯에 용병을 등록하면, 시스템은 용병의 스탯/장비/자질 태그와 미션의 요구 조건/위협/보상 구조를 상호 대조(Cross-Check)한다.
*   단, 표시되는 매칭 정보의 깊이는 반드시 `min(용병 분석 레벨, 미션 분석 레벨)` 규칙을 따른다.
*   세부 UI 연출 및 신호등 표시 로직은 `미션 카드 UI 및 매칭 시스템 메모.md`를 따른다.

## 2. 구역(Tier)별 생존/사망 판정 시스템

이 게임에서 미션 실패는 단순한 "게임 오버"나 "미션 재시작"을 의미하지 않는다. 
작전 도중 판정 실패로 탈출에 실패하거나 기권(조기 철수 패널티 지불)하지 못하고 요격당했을 경우, 용병의 목숨을 담보로 하는 **[사생결단 생존 판정]**이 기동한다.

### 2.1 생존 판정 공식 (Survival Formula)
미션 최종 탈출 실패 시, 해당 구역의 기본 생존율을 바탕으로 용병의 특정 능력치 및 보유 자질을 대조하여 영구 사망 혹은 생존(의체 훼손 귀환) 여부를 최종 연산한다.

$$\text{최종 생존 확률(\%)} = \text{구역별 기본 생존율} + (\text{구역별 핵심 생존 스탯} \times \text{스탯 가중치}) + \text{보유 적합 자질 보정} - \text{치명적 약점 자질 감산}$$

*   **스탯 가중치**: 구역별 핵심 생존 스탯의 `10%` 수치를 확률에 가산한다. (예: 해당 스탯이 `50`인 경우 `+5%` 보정)
*   **판정 결과**:
    *   **생존 성공 (의체 훼손)**: 용병이 목숨은 건져 귀환했으나, 신체/정신 상태에 치명적이고 영구적인 후유증 태그(예: `#산성_폐오염`, `#전뇌_단락_충격`)를 획득하고 장착 중인 임플란트 및 기어가 완전 파손(수리비 폭증)된다.
    *   **생존 실패 (영구 사망)**: 용병이 현장에서 살해되거나 전뇌가 완전히 구워져 사망한다. 이력서 파일은 영구 비활성화(`사망` 처리)되며, 소지 기어는 영구 유실된다.

### 2.2 구역별 생존 환경 및 판정 축 상세 설계

#### A. 하층 (Tier 3: 유앤존 및 산업 폐기물 지대)
*   **환경의 본질**: 법과 보호가 완전히 배제된 야생의 침수 구역. 무자비한 환경 독성 물질, 붕괴하는 폐격벽, 부패한 부랑자들의 날것 그대로의 살인적 기습.
*   **기본 생존율**: **60%**
*   **핵심 생존 스탯**: 
    *   **프레임 (Frame)**: 치명적인 독성 유출을 의체 독독 기능으로 버텨내거나 붕괴하는 격벽을 힘으로 지탱해 탈출.
    *   **펄스 (Pulse)**: 어둠 속에서 날아오는 불규칙한 낙하물 및 약탈자의 원초적 기습을 감지하고 회피.
*   **생존율 보정 자질**:
    *   `+15%` 보정 자질: `#배수구_작업`, `#벤트_적응`, `#슬럼_출신`, `#자가_회복력`
    *   `-20%` 감산 자질 (약점): `#폐질환` (유독가스 마시는 즉시 사망 판정), `#거구` (격벽 매몰 시 끼임사), `#소심함` (약탈자 기습 시 패닉 도주 불가)

#### B. 중층 (Tier 2: 콜 중층 — 4대 메가코프 계획도시)
*   **환경의 본질**: 삼엄한 치안 기업 K-Sec의 무자비한 법 집행, 골목마다 영역을 장악한 악독한 스트리트 갱단의 요격, 그리고 소란 유발 시 폭증하는 실시간 추격대.
*   **기본 생존율**: **40%**
*   **핵심 생존 스탯**:
    *   **사이퍼 (Cypher)**: 추격대의 추적 경로를 무선 교란하고 보안 카메라 망을 역스캔하여 우회 동선 개척.
    *   **쿨 (Cool)**: K-Sec에 포위되거나 신분 스캔을 당했을 때 위조 신분으로 평정심을 유지하며 탈출 구멍 확보.
*   **생존율 보정 자질**:
    *   `+15%` 보정 자질: `#시민_위장`, `#스캐너_기만`, `#라나다_단골`, `#기업_도주_경로`
    *   `-25%` 감산 자질 (약점): `#무장노출` (검문 적발 즉시 케이섹 집중사격 대상 등재), `#충동적` (K-Sec 가드와 불필요한 마찰 유발), `#기업_블랙리스트` (신원 대조 즉시 요격 명령 발동)

#### C. 상층 (Tier 1: 가든 지구 펜트하우스 및 기업 연구소)
*   **환경의 본질**: 금박을 입힌 정밀 살상 지대. 들키는 순간 1초 만에 살을 녹여버리는 인공 바이오 전멸 레이저, 숨소리조차 감지해 요격하는 정예 하이테크 기업 암살 조대, 그리고 흔적 자체를 우주에서 포맷해 버리는 무자비한 보안망.
*   **기본 생존율**: **20%**
*   **핵심 생존 스탯**:
    *   **쿨 (Cool)**: 극단의 정적 연회장 예법 기만 유지 및 요격 요원들의 심문 돌발 상황 무마.
    *   **사이퍼 (Cypher)**: 최고 등급 분자 센서 및 실시간 DNA 바이오메트릭 감시 게이트를 동적으로 교란.
*   **생존율 보정 자질**:
    *   `+20%` 보정 자질: `#유전자_기만`, `#단정한_인상`, `#플러스원_안내_가드`, `#기업_비밀연줄`
    *   `-30%` 감산 자질 (약점): `#슬럼_출신` (상층 출입 시 격식 결여 및 냄새 센서로 즉각 위조 신분 파열), `#소란_제조기` (소리 내는 즉시 정예 살상 드론 일점사 타겟팅)

## 3. 난이도별 노드 수 스케일링 계산법

미션의 실질 난이도(별 개수)는 체크하는 관문 노드의 총개수와 정비 리스크에 직관적으로 비례한다.

### 3.1 노드 수 및 피로도 누적 테이블
*   **노드 진행률 비례 공식**:
    $$\text{미션 노드 개수} = 3 + (\text{난이도 성(Star) 개수} - 1)$$
*   난이도가 높아질수록 판정 횟수가 누적되므로, 용병에게 매 노드마다 기본 부과되는 **`피로도(Fatigue)`**와 **`의체 내구도 마손도`**가 기하급수적으로 폭증한다.

| 난이도 (Difficulty) | 관문 노드 수 | 노드 통과 시 기본 피로도 누적량 | 권장 최소 주 스탯 | 예상 의체 손실 위험도 |
| :---: | :---: | :---: | :---: | :---: |
| **쉬움 (★☆☆☆☆)** | **3개** | 노드당 **+3** (총 9) | 스탯 `30` 대역 | 매우 안전 (마모 미미) |
| **보통 (★★☆☆☆)** | **4개** | 노드당 **+5** (총 20) | 스탯 `40` 대역 | 보통 (정비 필요 수준) |
| **어려움 (★★★☆☆)** | **5개** | 노드당 **+8** (총 40) | 스탯 `50` 대역 | 주의 (경미한 후유증 전이) |
| **치명적 (★★★★☆)** | **6개** | 노드당 **+12** (총 72) | 스탯 `60` 대역 | 위험 (의체 부품 일부 파열) |
| **파멸적 (★★★★★)** | **7개** | 노드당 **+18** (총 126) | 스탯 `75` 초과 | 극악 (귀환 시 사지 절단 및 전뇌 붕괴) |

## 4. 로컬 AI(Gemma 2) 하드보일드 브리핑 템플릿 변수 설계

시스템 알고리즘이 연산하여 결정한 "기계적 미션 정보"를 기반으로, 플레이어가 읽을 **하드보일드 풍의 의뢰 브리핑 줄글**을 로컬 AI가 생성할 수 있게 유도하는 표준 템플릿 스키마이다.

### 4.1 로컬 AI 입력 매개변수 (Input Parameters)
알고리즘은 로컬 AI API를 호출할 때 아래의 변수 데이터를 JSON 형태로 가공하여 프롬프트의 재료로 삽입한다.

```json
{
  "Mission_Type": "잠입 (Stealth)",
  "Tier_Level": "Tier 2 (중층 - 클럽가 서쪽 지하 2층)",
  "Required_Gear": "은닉형 보관 슬롯, 가시성 보통 이하 무기",
  "Target_Objective": "라 나다 소유의 클럽 엘 네온 VIP 금고 코어 칩 분리 강탈",
  "Tone_Style": "Hardboiled, Gritty, Cyberpunk Street Slang, 3-5 sentences"
}
```

### 4.2 로컬 AI 프롬프트 생성 템플릿 규칙
로컬 AI가 문맥을 일탈하거나 지나치게 밝은 톤의 줄글을 출력하지 않도록, 아래와 같은 제약(Few-shot 및 시스템 명령)을 고정 주입한다.

```text
[시스템 명령어]
너는 2080년대 디스토피아 사이버펑크 세계관의 무자비하고 냉소적인 10년 차 픽서다.
입력된 JSON 정보를 바탕으로, 플레이어(용병 관리자)가 현장 위험도를 스스로 추론할 수 있는 하드보일드 브리핑 줄글을 단 3~5문장으로 생성하라.

[제약 사항]
1. "Frame 스탯이 필요하다" 또는 "은닉 슬롯을 껴라"와 같은 기계적인 시스템 능력치를 문장 안에 직접 언급하지 마라.
2. 대신 환경적인 단서(예: "K-Sec 검색대가 칼같이 신체 스캔을 돌리고 있다", "드러난 군용 무기가 보이면 소란이 일어날 거다")로 대체하여 기만적으로 묘사하라.
3. 비속어나 지나치게 유치한 슬랭을 지양하고, 냉정하고 씁쓸한 거리의 어조(Street Tone)를 철저히 유지하라.

[출력 예시]
"클럽 엘 네온은 밤마다 라 나다 갱단놈들이 불법 약물을 풀고, 부패한 기업 회계사들이 판돈을 굴리는 중층 뒷골목의 지옥이지. 입구의 키쿠모토식 안면 식별 스캐너만 넘어가면, 안에는 온통 네온 불빛과 만취한 시민들로 엉망진창이야. VIP 룸 구석의 금고에 꼽혀 있는 암호 칩을 뽑아 와라. 라 나다 놈들에게 걸리는 순간, 도박장 셔터가 내려지고 기관총이 사방에서 불을 뿜을 테니 알아서 은밀하게 움직여."
```
